# PIKIT 시놀로지(NAS) 배포 및 자동 업데이트 가이드

파일 복사가 엄청나게 느린 이유는 폴더 내부에 있는 숨겨진 `.git` 폴더와 수만 개의 작은 파일로 이루어진 `node_modules` 폴더 때문입니다.
하지만 Docker를 사용해 배포할 때는 **`node_modules` 폴더를 복사할 필요가 전혀 없습니다!** (빌드 과정에서 알아서 설치됩니다.)

빠르게 배포하고 앞으로의 업데이트도 원버튼(혹은 자동)으로 처리하는 가장 좋은 방법을 안내해 드립니다.

---

## 1. 지금 당장 빠르게 옮기는 방법 (압축 방식)
수만 개의 파일을 복사하는 대신, 단 하나의 압축 파일로 만들어서 옮기면 1초 만에 전송됩니다.

1. 맥북에서 `PIKIT` 폴더 안의 `node_modules` 폴더를 제외하고 나머지 파일들만 선택하여 압축(`PIKIT.zip`)합니다.
2. 만들어진 `PIKIT.zip` 파일을 시놀로지 NAS의 대상 폴더(예: `/docker/pikit`)로 드래그 앤 드롭합니다.
3. 시놀로지 **File Station(파일 스테이션)** 에서 압축 파일을 우클릭하여 '여기에 압축 풀기'를 누릅니다.
4. 이후 앞서 안내해 드린 Container Manager(Docker)에서 프로젝트를 생성하여 실행하면 끝입니다.

---

## 2. 🌟 미래를 위한 최적의 방법 (Git을 통한 업데이트)
앞서 `git push`를 통해 코드를 원격 저장소(GitHub 등)에 업로드해 두셨습니다.
시놀로지 NAS가 직접 Git 저장소에서 코드를 다운로드(Pull) 받게 만들면, 맥북에서 NAS로 파일을 전송할 필요가 아예 없어집니다!

### [최초 1회 설정] NAS에 Git Clone 받기
시놀로지 제어판의 **작업 스케줄러(Task Scheduler)** 를 이용하면 SSH 접속 없이도 명령어를 실행할 수 있습니다.
1. 제어판 -> **작업 스케줄러** -> 생성 -> **예약된 작업** -> **사용자 정의 스크립트**
2. **다음을 위해 실행:** 사용자를 `root`로 변경
3. **작업 설정** 탭의 "사용자 정의 스크립트" 입력란에 아래 코드를 넣습니다.
   ```bash
   # 다운로드 받을 경로로 이동 (본인의 경로에 맞게 수정하세요)
   cd /volume1/docker
   
   # GitHub에서 소스 코드 다운로드 (저장소가 Private이라면 Personal Access Token 필요)
   git clone [개인 GitHub 또는 Git 저장소 주소.git] pikit
   ```
4. 저장 후 해당 작업을 우클릭하여 **실행**을 누르면, NAS가 알아서 코드를 다운로드 받습니다.
5. 이후 다운로드된 폴더를 Container Manager에서 프로젝트로 등록하여 실행시킵니다.

---

### [이후 업데이트 방법] 코드 수정 후 원클릭 배포
코드나 에셋 디자인이 변경되었을 때, 맥북과 NAS 양쪽에서 다음 작업만 해주면 됩니다.

**A. 맥북(로컬)에서 할 일:**
코드를 수정하고 프론트엔드 빌드 후 Git에 올립니다.
```bash
npm run build:wallet
git add .
git commit -m "게임 밸런스 및 UI 업데이트"
git push
```

**B. 시놀로지 NAS에서 할 일:**
다시 한번 제어판 -> **작업 스케줄러**에 다음과 같은 "업데이트 전용 스크립트"를 만들어 둡니다. (사용자: `root`)
```bash
# PIKIT 폴더로 이동
cd /volume1/docker/pikit

# 최신 코드 깃허브에서 가져오기
git pull origin main

# Docker 컨테이너 다시 빌드 및 재시작 적용
docker-compose down
docker-compose up -d --build
```
업데이트가 필요할 때마다 시놀로지 작업 스케줄러에서 이 스크립트만 **실행(Run)** 해주면 자동으로 최신 코드가 적용되고 서버가 재시작됩니다!
