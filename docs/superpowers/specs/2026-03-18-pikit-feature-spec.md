<a name="epic1"></a>
## ✦ [EPIC 1] 게임 접속 및 관전 서비스

### ❖ 1. 목적 및 배경

- **배경**: 진입 장벽을 낮춰 더 많은 잠재 유저가 게임을 경험하도록 해야 한다. 로그인 없이도 게임의 재미를 먼저 확인하고, 자연스럽게 참여를 유도할 수 있다.
- **목적**: 설치나 로그인 없이 브라우저 URL 접속만으로 실시간 게임 화면을 관전할 수 있게 하여, 신규 유저 유입 및 전환율을 높인다.

### ❖ 2. User Story & 상세 플로우

- **[STORY 1-1]** 비로그인 유저는 게임 체험을 위해 URL 접속만으로 게임 화면을 볼 수 있어야 한다.
  1. 유저가 브라우저에 URL을 입력하고 접속한다.
  2. 시스템은 브라우저에서 바로 게임 화면을 로드한다 (별도 설치 불필요).
  3. 유저는 로그인 없이도 실시간으로 진행 중인 게임(블록, 곡괭이, 폭발 이펙트)을 관전한다.

- **[STORY 1-2]** 비로그인 유저는 게임 현황 파악을 위해 실시간 게임 정보를 확인할 수 있어야 한다.
  1. 화면 상단 HUD에 **현재 접속 중인 플레이어 수**가 표시된다.
  2. **현재 활성화된 전체 곡괭이 수**가 표시된다.
  3. **현재 채굴 깊이(m)**가 표시된다.
  4. 다른 플레이어가 고가치 블록을 파괴하면 화면에 축하 배너 알림이 자동으로 표시된다.
  5. 잭팟 블록이 등장하면 전체 화면에 특별 알림이 표시된다.

### ❖ 3. 정책 및 데이터 정의

- **기본 정책**
  - 비로그인 관전 유저도 게임 상태를 실시간으로 수신한다 (초당 20회 브로드캐스트).
  - 관전 유저는 구매/채팅 등 액션이 불가하며, 시도 시 로그인 요청 안내가 표시된다.

### ❖ 4. 의존 영역 및 영향도

- **연동 시스템**: WebSocket(Socket.IO) 실시간 브로드캐스트
- **관련 기능**: EPIC 2 (로그인), EPIC 5 (곡괭이 구매), EPIC 12 (채팅) — 비로그인 시 모두 잠금 상태

<a name="epic2"></a>
## ✦ [EPIC 2] 로그인 서비스

### ❖ 1. 목적 및 배경

- **배경**: 게임 참여(구매, 채팅, 퀘스트, 입출금)를 위해서는 신원 확인이 필요하다. 두 가지 진입 경로(닉네임 / Web3 지갑)를 제공해 접근성과 보안을 모두 충족한다.
- **목적**: 간편한 로그인을 통해 참여 마찰을 최소화하고, 지갑 연결로 블록체인 기능(입출금, 온체인 퀘스트)을 활성화한다.

### ❖ 2. User Story & 상세 플로우

- **[STORY 2-1]** 유저는 빠른 게임 참여를 위해 닉네임만 입력해 로그인할 수 있어야 한다.
  1. 유저가 닉네임 입력창에 이름을 입력한다.
  2. 시스템은 닉네임을 검증한다 (최대 12자, 특수문자 제거).
  3. 검증 통과 시 로그인 처리되고, 10,000 크레딧이 자동 지급된다.
  4. 유저는 게임 화면에서 구매·채팅이 가능한 상태가 된다.

- **[STORY 2-2]** 유저는 블록체인 기능 사용을 위해 Web3 지갑을 연결해 로그인할 수 있어야 한다.
  1. 유저가 '지갑 연결' 버튼을 클릭한다.
  2. MetaMask(또는 RainbowKit 지원 지갑) 연결 팝업이 표시된다.
  3. 유저가 지갑 연결에 동의하면, 시스템이 서명 메시지를 요청한다.
  4. 유저가 서명하면 시스템은 서명을 검증해 본인 확인을 완료한다.
  5. 로그인 처리되고 10,000 크레딧이 자동 지급된다.
  6. 지갑 주소(`0x1234...abcd` 형식)가 닉네임으로 사용된다.
  7. USDC 입출금, 온체인 퀘스트 기능이 활성화된다.

- **[STORY 2-3]** 유저는 계정 전환 또는 게임 종료를 위해 로그아웃할 수 있어야 한다.
  1. 유저가 '내 정보' 팝업에서 로그아웃 버튼을 클릭한다.
  2. `정말 로그아웃 하시겠습니까?` 확인 팝업이 표시된다.
  3. 확인 클릭 시 로그아웃 처리 후 페이지가 새로고침된다.
  4. 취소 클릭 시 팝업이 닫히고 게임이 그대로 유지된다.

### ❖ 3. 정책 및 데이터 정의

- **기본 정책**
  - 닉네임 최대 길이: **12자**
  - 닉네임 금지 문자: `< > & " ' /`
  - 닉네임 미입력 시 기본값: `Player`
  - 초기 지급 크레딧: **10,000**
  - 이미 로그인 상태에서 재로그인 시도는 거부된다
  - 지갑 연결 시 닉네임 길이 제한: **13자** (지갑 단축 주소 `0x1234...abcd` = 13자)
- **예외/에러 상황**
  - 서명 검증 실패 → `서명 검증에 실패했습니다` 오류 표시, 로그인 거부
  - 이미 로그인 상태에서 재시도 → `이미 참여 중입니다` 오류 표시
  - 브라우저 창 닫기/연결 끊김 → 자동 세션 종료 (크레딧 저장 안 됨, 세션 기반)
- **유저 데이터 정의**
  - 닉네임 (필수) 또는 지갑 주소 (지갑 로그인 시 자동)

### ❖ 4. 의존 영역 및 영향도

- **연동 시스템**: Ethereum 서명 검증(ethers.js), RainbowKit/MetaMask SDK
- **관련 기능**: EPIC 3 (크레딧 지급), EPIC 5/6 (구매), EPIC 10 (퀘스트), EPIC 14 (입출금) — 로그인 이후 활성화

<a name="epic3"></a>
## ✦ [EPIC 3] 크레딧(게임 내 화폐) 서비스

### ❖ 1. 목적 및 배경

- **배경**: 게임 내 모든 구매 및 보상은 '크레딧'이라는 가상 화폐로 처리된다. 실제 돈(USDC)과도 연동된다.
- **목적**: 크레딧 흐름(지급·소비·획득·표시)을 명확히 정의해 게임 경제 시스템의 신뢰성을 확보한다.

### ❖ 2. User Story & 상세 플로우

- **[STORY 3-1]** 유저는 게임 시작을 위해 로그인 시 초기 크레딧을 무료로 받을 수 있어야 한다.
  1. 유저가 로그인(닉네임 또는 지갑)에 성공한다.
  2. 시스템이 자동으로 10,000 크레딧을 지급한다.
  3. 화면 상단 HUD에 잔액이 즉시 반영된다.

- **[STORY 3-2]** 유저는 크레딧 부족 시 무료 크레딧을 추가 충전할 수 있어야 한다.
  1. 유저가 '+ 크레딧 충전' 버튼을 클릭한다.
  2. 시스템은 30초 내 요청 횟수를 확인한다.
  3. 5회 미만이면 10,000 크레딧을 즉시 지급하고 잔액을 갱신한다.
  4. 5회 초과이면 요청을 무시한다 (별도 오류 없음).

- **[STORY 3-3]** 유저는 현재 상태 파악을 위해 크레딧 잔액을 항상 확인할 수 있어야 한다.
  1. 화면 상단 HUD에 현재 크레딧 잔액이 항상 표시된다.
  2. 구매 또는 보상 발생 즉시 잔액이 실시간 갱신된다.

### ❖ 3. 정책 및 데이터 정의

- **기본 정책**
  - 초기 지급 크레딧: **10,000**
  - 무료 충전 단위: 1회 **10,000 크레딧**
  - 무료 충전 한도: **30초당 최대 5회**
  - 크레딧은 정수로만 표기된다
  - 크레딧은 세션 기반 — 로그아웃 시 사라짐 (USDC 입금으로만 영속 가능)
- **예외/에러 상황**
  - 충전 한도 초과 시 무시 처리 (에러 메시지 없음)

## ✦ [EPIC 4] 필드 선택 서비스

### ❖ 1. 목적 및 배경

- **배경**: 유저의 성향과 리스크 선호도가 다르다. 소액으로 플레이하고 싶은 유저와 대규모 수익을 노리는 유저 모두를 수용할 수 있어야 한다.
- **목적**: Normal / Hardcore 두 가지 필드를 운영해 다양한 플레이 스타일을 지원하고, 동시 접속자를 분산 수용한다.

### ❖ 2. User Story & 상세 플로우

- **[STORY 4-1]** 유저는 플레이 스타일 선택을 위해 Normal 또는 Hardcore 필드를 선택할 수 있어야 한다.
  1. 화면에서 필드 토글 스위치(Normal ↔ Hardcore)를 클릭한다.
  2. Hardcore를 처음 선택할 경우, `Hardcore 모드: 비용과 보상이 10배입니다. 계속하시겠습니까?` 확인 팝업이 표시된다.
  3. 확인 클릭 시 Hardcore 필드로 전환된다.
  4. 취소 클릭 시 Normal 필드 상태를 유지한다.
  5. 이후 전환 시 `다시 묻지 않기` 체크박스를 선택했다면 팝업 없이 바로 전환된다.
  6. 전환 완료 시 해당 필드의 게임 화면으로 전환되며, 이전 필드의 곡괭이는 제거된다.

### ❖ 3. 정책 및 데이터 정의

- **기본 정책**

  | 필드명 | 비용 배율 | 보상 배율 |
  |--------|-----------|-----------|
  | Normal | 1x | 1x |
  | Hardcore | 10x | 10x |

  - 같은 필드의 플레이어들끼리만 같은 게임 화면을 공유한다
  - 필드 전환 시 이전 필드의 활성 곡괭이는 즉시 제거된다
  - `다시 묻지 않기` 설정은 브라우저 로컬에 저장된다

- **예외/에러 상황**
  - 로그인 없이 필드 전환 시도 → 로그인 요청 안내 표시

<a name="epic5"></a>
## ✦ [EPIC 5] 곡괭이 구매 및 채굴 서비스

### ❖ 1. 목적 및 배경

- **배경**: 곡괭이 구매가 PIKIT의 핵심 게임플레이다. 유저는 크레딧을 투자해 곡괭이를 사고, 곡괭이가 블록을 파괴하면 크레딧을 획득한다.
- **목적**: 4종의 곡괭이 각각에 뚜렷한 개성과 전략적 선택지를 부여해 반복 구매 동기를 만든다. v4.7 밸런스 기준으로 평균 ROI 47~52%를 유지한다.

### ❖ 2. User Story & 상세 플로우

- **[STORY 5-1]** 유저는 구매 결정을 위해 각 곡괭이의 스펙과 가격을 확인할 수 있어야 한다.
  1. 유저가 화면의 '상점(Shop)' 버튼을 클릭한다.
  2. Shop 패널이 열리며 4종 곡괭이 목록이 표시된다.
  3. 각 항목에 이름, 가격, 공격력, 지속시간, 특수 효과 설명이 표시된다.
  4. 현재 필드 배율이 가격에 반영된다 (Hardcore에서는 10배 가격).
  5. 현재 내 활성 곡괭이 수 (`예: Active Pickaxes: 1/3`)가 표시된다.

- **[STORY 5-2]** 유저는 채굴 참여를 위해 원하는 곡괭이를 구매할 수 있어야 한다.
  1. 유저가 Shop 패널에서 구매할 곡괭이를 클릭한다.
  2. 시스템은 잔액과 활성 곡괭이 수를 확인한다.
  3. 조건 충족 시 크레딧이 차감되고 곡괭이가 화면 상단의 빈 열(column)에서 낙하를 시작한다.
  4. 조건 미충족 시 오류 메시지가 표시된다.

- **[STORY 5-3]** 유저는 크레딧 획득을 위해 곡괭이가 블록을 파괴하면 자동으로 보상을 받을 수 있어야 한다.
  1. 낙하 중인 곡괭이가 블록에 충돌하면 블록의 HP를 공격력만큼 감소시킨다.
  2. 블록 HP가 0 이하가 되면 블록이 파괴된다.
  3. 블록 보상(고정 또는 랜덤)이 즉시 유저 잔액에 더해진다.
  4. 콤보가 적용된 경우 보상에 배율이 곱해진다.
  5. 블록 파괴 후 곡괭이는 블록에서 튕겨 나간다.
  6. 곡괭이는 수명이 남아 있는 동안 계속 낙하 및 충돌을 반복한다.

- **[STORY 5-4]** 유저는 곡괭이 투자 결과를 확인하기 위해 곡괭이 만료 시 결과 알림을 받을 수 있어야 한다.
  1. 곡괭이 지속시간이 종료되면 해당 곡괭이가 필드에서 자동 제거된다.
  2. 유저에게 알림이 전달된다:
     - 곡괭이 종류
     - 해당 곡괭이로 총 획득한 크레딧
     - 해당 곡괭이로 파괴한 총 블록 수
  3. 활성 곡괭이 카운트가 1 감소한다 (예: 2/3 → 1/3).

### ❖ 3. 정책 및 데이터 정의

- **기본 정책**
  - 동시 최대 활성 곡괭이: 플레이어당 **3개** (필드 기준)
  - 곡괭이는 구매 시 빈 열에서 낙하 시작 (빈 열 없을 시 랜덤 열)
  - 한 번의 충돌에서 **1개 블록에만** 피해 적용

  - 미로그인 상태에서 Shop 클릭 → 로그인 요청 안내 표시

<a name="epic6"></a>
## ✦ [EPIC 6] TNT 구매 및 폭발 서비스

### ❖ 1. 목적 및 배경

- **배경**: 곡괭이만으로는 고HP 블록을 빠르게 파괴하기 어렵다. TNT는 범위 피해를 통해 순간적으로 다수의 블록을 파괴할 수 있는 전략 아이템이다.
- **목적**: TNT를 통해 잠재적인 고수익 순간을 만들고, 다양한 플레이 전략(잭팟 블록 노림, 고밀도 구역 청소)을 제공한다.

### ❖ 2. User Story & 상세 플로우

- **[STORY 6-1]** 유저는 범위 공격을 위해 TNT를 구매하고 필드에 투입할 수 있어야 한다.
  1. 유저가 Shop 패널에서 TNT 항목을 클릭한다.
  2. 시스템이 잔액을 확인한다.
  3. 잔액 충분 시 크레딧을 차감하고 TNT가 화면 상단 랜덤 위치에서 낙하를 시작한다.
  4. 잔액 부족 시 오류 메시지가 표시된다.

- **[STORY 6-2]** 유저는 크레딧 획득을 위해 TNT가 블록에 닿아 폭발하면 범위 내 보상을 자동으로 받을 수 있어야 한다.
  1. 낙하 중인 TNT가 블록에 닿는 순간 폭발한다 (타이머가 아닌 충돌 기반).
  2. 시스템은 폭발 범위(가로 5칸 × 세로 4칸) 내 모든 블록에 피해를 적용한다.
  3. TNT 저항 블록은 40% 피해만 받는다 (즉시 파괴 불가).
  4. 파괴된 모든 블록의 보상이 합산되어 즉시 유저 잔액에 지급된다.
  5. 화면에 폭발 이펙트가 시각적으로 표시된다.

### ❖ 3. 정책 및 데이터 정의

- **기본 정책**

  | 항목 | Normal 필드 | Hardcore 필드 |
  |------|-------------|--------------|
  | TNT 가격 | **8,000 크레딧** | **80,000 크레딧** |
  | TNT 공격력 | 30 | 30 |
  | 폭발 범위 | 가로 5칸(좌우 ±2), 세로 4칸 | 동일 |
  | 개수 제한 | 없음 | 없음 |

  - TNT 저항 블록: Diamond Block, Gold Block, Emerald Block, Jackpot Block — 피해 **40%**만 적용
  - 일반 블록: 공격력 30으로 대부분 1타 파괴
  - 총 보상이 **5,000 크레딧 이상**이면 잭팟 피드에 노출

- **예외/에러 상황**
  - 잔액 부족 → `잔액이 부족합니다` 오류 표시

### ❖ 4. 의존 영역 및 영향도

- **관련 기능**: EPIC 3 (크레딧 차감/지급), EPIC 7 (블록 보상), EPIC 9 (잭팟 진행도 카운트)

## ✦ [EPIC 7] 블록 보상 및 알림 서비스

### ❖ 1. 목적 및 배경

- **배경**: 블록 종류와 보상 정보를 유저에게 명확히 전달해야 전략적 플레이가 가능하다. 희귀 블록 등장 알림은 흥분감을 높이고 참여를 유도한다.
- **목적**: 블록 가이드와 실시간 알림으로 유저가 게임 경제를 이해하고, 희귀 이벤트를 놓치지 않도록 한다.

### ❖ 2. User Story & 상세 플로우

- **[STORY 7-1]** 유저는 전략 수립을 위해 모든 블록의 보상과 등장 확률을 확인할 수 있어야 한다.
  1. 유저가 메뉴 패널을 열면 '블록 가이드' 섹션이 표시된다.
  2. 각 블록의 이름, 보상, 등장 확률이 목록으로 나열된다.
  3. 잭팟 블록은 '특별 조건'으로 표시된다.

- **[STORY 7-2]** 유저는 고가치 기회를 포착하기 위해 희귀 블록이 필드에 등장하면 즉시 알림을 받을 수 있어야 한다.
  1. 다이아몬드/골드 블록이 새로 생성되면 화면 상단에 스폰 알림 배너가 표시된다.
  2. 배너에는 블록 이름과 최대 보상 크레딧이 포함된다.
  3. 4초 후 배너가 자동으로 사라진다.

- **[STORY 7-3]** 전체 유저는 흥분감 유지를 위해 누군가 고가치 블록을 파괴했을 때 알림을 받을 수 있어야 한다.
  1. 플레이어가 1,000 크레딧 이상 블록을 파괴하면 잭팟 피드에 알림이 표시된다.
  2. 다이아몬드/골드 블록 파괴 시 별도 화면 배너 알림이 해당 필드 전체 유저에게 표시된다.
  3. 잭팟 블록 파괴 시 15초간 대형 배너 알림이 표시된다.

### ❖ 3. 정책 및 데이터 정의

## ✦ [EPIC 8] 콤보 시스템 서비스

### ❖ 1. 목적 및 배경

- **배경**: 단순 반복 구매만으로는 게임의 재미가 줄어든다. 연속 파괴에 보너스를 부여해 몰입감과 스릴을 높인다.
- **목적**: 콤보 배율로 적극적인 플레이를 보상하고, 곡괭이를 여러 개 운용할 동기를 만든다.

### ❖ 2. User Story & 상세 플로우

- **[STORY 8-1]** 유저는 더 높은 보상을 위해 연속으로 블록을 파괴하면 콤보 배율을 쌓을 수 있어야 한다.
  1. 곡괭이가 블록을 파괴한다.
  2. 시스템은 마지막 파괴로부터 **2초 이내**에 추가 파괴가 발생하면 콤보를 1 증가시킨다.
  3. 콤보 단계에 따른 배율이 해당 블록 보상에 즉시 적용된다.
  4. 2초가 지나도록 파괴가 없으면 콤보가 1(기본)로 초기화된다.

### ❖ 3. 정책 및 데이터 정의

- **기본 정책**
  - 콤보 타임아웃: **2초**
  - 콤보는 **곡괭이 단위**로 개별 추적 (다른 곡괭이/다른 플레이어의 콤보와 무관)
  - 콤보 배율 최대: **1.5x**

  | 콤보 횟수 | 보상 배율 |
  |-----------|-----------|
  | 0 ~ 2회 | **1.0x** |
  | 3 ~ 5회 | **1.05x** |
  | 6 ~ 9회 | **1.1x** |
  | 10 ~ 14회 | **1.2x** |
  | 15 ~ 24회 | **1.35x** |
  | 25회 이상 | **1.5x** |

  - Hardcore 필드에서는 콤보 배율(최대 1.5x)과 필드 배율(10x)이 중첩 적용

## ✦ [EPIC 9] 잭팟 시스템 서비스

### ❖ 1. 목적 및 배경

- **배경**: 대부분의 세션은 평균적인 수익으로 끝난다. 잭팟 시스템은 '대박의 꿈'을 제공해 유저가 계속 플레이하도록 동기를 부여한다.
- **목적**: 희귀하고 어려운 조건을 만족할 때만 잭팟 블록이 등장하도록 설계해, 등장 자체만으로도 게임 내 이벤트가 되는 특별한 경험을 만든다.

### ❖ 2. User Story & 상세 플로우

- **[STORY 9-1]** 유저는 기회 포착을 위해 잭팟 블록이 필드에 등장하면 즉시 전체 알림을 받을 수 있어야 한다.
  1. 잭팟 등장 조건 3가지가 모두 충족되면 시스템이 잭팟 블록을 생성한다.
  2. 해당 필드의 모든 유저에게 `🏆 잭팟 블록이 등장했습니다!` 알림이 즉시 표시된다.

- **[STORY 9-2]** 유저는 대규모 보상 획득을 위해 잭팟 블록을 파괴하면 250,000 크레딧을 받을 수 있어야 한다.
  1. 유저의 곡괭이가 잭팟 블록(HP 300)에 피해를 입힌다.
  2. 블록 HP가 0이 되면 최종 타격을 한 유저에게 250,000 크레딧이 지급된다.
  3. 해당 필드의 모든 유저에게 `[플레이어 이름]이(가) 잭팟 블록을 파괴했습니다! +250,000` 알림이 15초간 표시된다.
  4. 잭팟 카운터가 초기화되고 다음 잭팟 대기 상태로 전환된다.

### ❖ 3. 정책 및 데이터 정의

- **잭팟 블록 등장 조건 (3가지 모두 충족 필요)**
  1. 현재 필드 활성 플레이어 **10명 이상**
  2. 이전 잭팟 이후 소비된 크레딧 합계 **1,500,000 이상**
  3. 블록 생성 시 **0.05%** 확률 통과

- **기본 정책**
  - 잭팟 블록 보상: **250,000 크레딧** (고정)
  - 잭팟 블록 HP: **300** (TNT 저항, 40% 피해만 적용)
  - 필드당 동시에 최대 **1개**만 존재
  - 잭팟 블록이 화면 밖으로 스크롤되어 사라지면 잭팟 기회 소멸

## ✦ [EPIC 10] 퀘스트 서비스

### ❖ 1. 목적 및 배경

- **배경**: 반복 플레이에 목표 의식을 부여하지 않으면 유저 이탈이 빠르다. 퀘스트는 단기 목표를 제공해 세션 내 체류 시간을 늘린다.
- **목적**: 10종의 달성 가능한 퀘스트를 제공하고, 블록체인에 완료 기록을 남겨 성취감과 실질적 가치를 동시에 제공한다.

### ❖ 2. User Story & 상세 플로우

- **[STORY 10-1]** 유저는 달성 목표 확인을 위해 퀘스트 목록과 현재 진행 현황을 볼 수 있어야 한다.
  1. 유저가 '퀘스트' 버튼을 클릭한다.
  2. 퀘스트 패널이 열리며 10종 퀘스트 목록이 표시된다.
  3. 각 항목에 이름, 달성 조건, 현재 진행치/목표치, 달성률(%) 바가 표시된다.
  4. 목표 달성 시 '온체인 완료' 버튼이 활성화된다.
  5. 이미 완료된 퀘스트는 ✅ 아이콘과 함께 완료 상태로 표시된다.

- **[STORY 10-2]** 유저는 영구 기록을 남기기 위해 달성한 퀘스트를 블록체인에 기록하고 완료 처리할 수 있어야 한다.
  1. 유저가 '온체인 완료' 버튼을 클릭한다.
  2. 시스템이 지갑 연결 여부와 퀘스트 컨트랙트 배포 여부를 확인한다.
  3. 조건 충족 시 유저 지갑에 트랜잭션 서명 요청이 발생한다.
  4. 유저가 서명하면 버튼이 `검증 중...`으로 전환되고 트랜잭션이 전송된다.
  5. 트랜잭션이 온체인(Sepolia)에서 확인되면 서버가 검증 후 해당 퀘스트를 완료 처리한다.
  6. 퀘스트 목록에서 해당 퀘스트가 ✅ 완료 상태로 갱신된다.

### ❖ 3. 정책 및 데이터 정의

  | 4 | Gold Rush | 금 블록 5개 파괴 |

  | 6 | Shopping Spree | 곡괭이 10개 구매 |

  - 게임 서버가 목표 달성 여부를 독립적으로 검증 (클라이언트 조작 불가)

  - **동일 퀘스트 중복 완료 불가**
  - Ethereum Sepolia 테스트넷 기준 동작

- **예외/에러 상황**
  - 지갑 미연결 → `지갑을 먼저 연결해주세요` 오류
  - 퀘스트 컨트랙트 미배포 → `퀘스트 컨트랙트가 아직 배포되지 않았습니다` 오류
  - 트랜잭션 서명 거부 → 오류 메시지 표시, 버튼 재활성화
  - 잘못된 네트워크 → 트랜잭션 실패, `올바른 네트워크(Sepolia)를 확인해주세요` 오류
  - 게임 내 목표 미달성 상태 → 서버가 완료 처리 거부
  - 이미 온체인 완료된 퀘스트 재시도 → `이미 완료된 퀘스트입니다` 오류

### ❖ 4. 의존 영역 및 영향도

- **연동 시스템**: Ethereum Sepolia 블록체인, 퀘스트 스마트 컨트랙트, ethers.js
- **관련 기능**: EPIC 2 (지갑 로그인 필수), EPIC 5 (블록 파괴 카운트), EPIC 6 (TNT 구매 카운트)

## ✦ [EPIC 11] 리더보드 서비스

### ❖ 1. 목적 및 배경

- **목적**: 실시간 순이익 순위를 통해 경쟁 동기를 부여하고, 상위권 유저들의 플레이를 참고하도록 유도해 체류 시간을 늘린다.

### ❖ 2. User Story & 상세 플로우

  1. 유저가 '리더보드' 버튼을 클릭한다.
  2. 리더보드 모달이 열리며 상위 10명의 이름과 순이익이 표시된다.
  3. 순위는 자동으로 2초마다 갱신된다.
  4. 모달 외부 클릭 또는 닫기(×) 버튼으로 닫을 수 있다.

### ❖ 3. 정책 및 데이터 정의

- **기본 정책**
  - 표시 기준: **순이익** = 총 획득 크레딧 - 총 소비 크레딧
  - 내림차순 정렬, 최대 **10명** 표시
  - 갱신 주기: **2초**
  - 1위: 금색, 2위: 은색, 3위: 동색 표시
  - 순이익 양수 → 녹색, 음수 → 빨간색
  - 플레이어 없음 → `아직 플레이어가 없습니다` 메시지
  - 리더보드는 필드별로 독립 (Normal/Hardcore 별도)

## ✦ [EPIC 12] 채팅 서비스

### ❖ 2. User Story & 상세 플로우

  2. 유저가 메시지를 입력하고 Enter 키 또는 전송 버튼을 클릭한다.

  5. 채팅창에 발신자 이름과 메시지가 표시된다.

### ❖ 3. 정책 및 데이터 정의

- **기본 정책**
  - 채팅 전송: **로그인 유저만** 가능
  - 최대 메시지 길이: **200자**
  - 전송 쿨다운: **1초** (초당 1개 제한, 도배 방지)
  - 채팅창 최대 유지 메시지 수: **100개** (초과 시 오래된 것 삭제)
  - 채팅은 **필드별 격리** — Normal과 Hardcore 채팅 분리
  - Escape 키로 채팅창 닫기 가능
  - 플레이어 입장 시 `[이름]이(가) 입장했습니다` 시스템 메시지 자동 표시
  - 플레이어 퇴장 시 `[이름]이(가) 퇴장했습니다` 시스템 메시지 자동 표시

- **예외/에러 상황**

<a name="epic14"></a>
## ✦ [EPIC 14] USDC 입출금 서비스

### ❖ 1. 목적 및 배경

- **배경**: 게임 내 크레딧을 실제 자산(USDC)과 연결할 수 있어야 게임의 실질적 가치가 생긴다. 이것이 일반 게임과 구분되는 핵심 차별점이다.
- **목적**: USDC 입금으로 크레딧을 충전하고, 충분한 크레딧이 쌓이면 USDC로 출금할 수 있게 해 실제 수익 창출의 가능성을 제공한다.

### ❖ 2. User Story & 상세 플로우

- **[STORY 14-1]** 유저는 크레딧 충전을 위해 USDC를 게임 크레딧으로 입금할 수 있어야 한다.
  1. 유저가 '내 정보' 팝업에서 입금(Deposit) 탭을 선택한다.
  2. 입금할 USDC 금액을 정수로 입력한다.
  3. 수령 예상 크레딧이 실시간으로 미리 표시된다 (`1 USDC = 10,000 크레딧`).
  4. 유저가 '입금' 버튼을 클릭한다.
  5. **1단계**: USDC 사용 승인(Approve) 트랜잭션 서명이 지갑에서 요청된다.
  6. 승인 트랜잭션이 온체인에서 확인된다.
  7. **2단계**: 입금(Deposit) 트랜잭션 서명이 지갑에서 요청된다.
  8. 입금 트랜잭션이 온체인에서 확인되면 서버가 크레딧을 유저에게 지급한다.
  9. 게임 잔액이 즉시 갱신된다.

- **[STORY 14-2]** 유저는 수익 실현을 위해 보유 크레딧을 USDC로 출금할 수 있어야 한다.
  1. 유저가 '내 정보' 팝업에서 출금(Withdraw) 탭을 선택한다.
  2. 출금할 USDC 금액을 정수로 입력한다.
  3. 차감될 크레딧이 실시간으로 미리 표시된다 (`1 USDC 출금 = 10,500 크레딧 차감`).
  4. 유저가 '출금' 버튼을 클릭한다.
  5. 잔액이 충분하면 출금 트랜잭션 서명이 지갑에서 요청된다.
  6. 트랜잭션이 온체인에서 확인되면 서버가 크레딧을 차감하고 USDC가 지갑으로 전송된다.
  7. 게임 잔액이 즉시 갱신된다.

### ❖ 3. 정책 및 데이터 정의

- **기본 정책**
  - 입금 비율: **1 USDC = 10,000 크레딧**
  - 출금 비율: **1 USDC = 10,500 크레딧 차감** (출금 수수료 5% 포함)
  - 입금/출금 단위: **정수 USDC만** 허용 (소수점 불가)
  - 동일 트랜잭션 해시로 **중복 입금/출금 불가** (리플레이 공격 방지)
  - 트랜잭션 발신자 주소가 로그인 지갑 주소와 **반드시 일치**해야 함
  - 네트워크: **Ethereum Sepolia 테스트넷**

- **예외/에러 상황**
  - 지갑 미연결 → `지갑을 먼저 연결해주세요` 오류
  - 볼트 컨트랙트 미배포 → `볼트 컨트랙트가 아직 배포되지 않았습니다` 오류
  - 입금 중 트랜잭션 거부 → 오류 메시지, 크레딧 미지급
  - 출금 시 잔액 부족 → `크레딧이 부족합니다. [N]크레딧 필요` 오류, 출금 거부
  - 출금 시 트랜잭션 거부 → 오류 메시지, 크레딧 미차감
  - 이미 처리된 트랜잭션 해시 재사용 → `이미 처리된 트랜잭션입니다` 오류

### ❖ 4. 의존 영역 및 영향도

- **연동 시스템**: Ethereum Sepolia 블록체인, PIKIT Vault 스마트 컨트랙트, USDC 컨트랙트(ERC-20), ethers.js
- **관련 기능**: EPIC 2 (지갑 로그인 필수), EPIC 3 (크레딧 잔액 갱신)

## ✦ [EPIC 15] 시스템(하우스) 곡괭이 서비스

### ❖ 1. 목적 및 배경

- **배경**: 게임 수익 구조(하우스 엣지)를 유지하기 위해 시스템이 직접 블록을 채굴하는 '하우스 곡괭이'가 필요하다. 이를 통해 플레이어 수가 적어도 하우스가 일정한 수익을 확보한다.
- **목적**: PIKIT이라는 시스템 곡괭이를 항상 필드에 운영해 목표 하우스 엣지(5인 기준 약 55%)를 유지한다.

### ❖ 2. User Story & 상세 플로우

- **[STORY 15-1]** 시스템은 하우스 엣지 유지를 위해 PIKIT 곡괭이를 항상 필드에서 자동 운영해야 한다.
  1. 게임 서버 시작 시 PIKIT 곡괭이가 자동으로 필드에 1개 배치된다.
  2. PIKIT 곡괭이가 제거되면 5초 이내에 새 PIKIT 곡괭이가 자동 생성된다.
  3. PIKIT 곡괭이는 플레이어 곡괭이와 동일하게 블록과 충돌하며 피해를 입힌다.
  4. 단, PIKIT 곡괭이가 블록을 파괴해도 어떤 플레이어에게도 보상이 지급되지 않는다.

### ❖ 3. 정책 및 데이터 정의

- **기본 정책**

  | 항목 | 값 | 비고 |
  |------|----|------|
  | 이름 | PIKIT | 하우스 전용 |
  | 공격력 | 5 | |
  | 크기 | 1.5배 (블록 기준) | 비주얼 임팩트, 멀티컬럼 |
  | 중력 배율 | 0.3배 | 매우 느린 낙하 (부유하듯) |
  | 속도 배율 | 0.1배 | 거의 정지에 가까운 이동 |
  | 수명 | 무한 | 만료 없음 |
  | 블록당 조우 빈도 | ~0.65회/초 | |
  | 보상 귀속 | 없음 | 하우스 수익으로 처리 |
  | 숍 노출 | 없음 | 플레이어 구매 불가 |
  | 색상 | 마젠타(#FF00FF) | 식별용 |

- **하우스 엣지 목표 (v4.7 기준):**

  | 플레이어 수 | 목표 | 실제 |
  |------------|------|------|
  | 5인 | 55% | ~55.3% ✅ |
  | 10인 | 54% | ~53.9% ✅ |
