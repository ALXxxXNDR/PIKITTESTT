# PIKIT v4.8 Balance Design Spec

> 작성일: 2026-03-23 | 브레인스토밍 결과 → 구현 명세

---

## 목표

1. **HE(하우스 엣지) 52~53%** — 2~30명 플레이어 구간 기준, 30~100명도 동일 범위
2. **최소 보상 바닥 보장(B안)** — 곡괭이 1개 기준 항상 일부 크레딧 회수 (0 회수 없음)
3. **동적 잭팟 풀** — 하우스 수익 기반 적립, 고정금액 폐지, 레이드 요소 추가
4. **어댑티브 시스템 곡괭이** — 최대 4개, 솔로 시 약화 모드, 100명까지 HE 일정 유지

## 변경 1: 어댑티브 시스템 곡괭이

### 1-1 모드 정의

| 모드 | 트리거 | scale | speedMult | gravityMult | 색상 | steal율(참고) |
|------|--------|-------|-----------|-------------|------|----------------|
| **휴식(Weak)** | 활성 플레이어 곡괭이 ≤ 3개 | 0.8 | 0.05 | 0.5 | 회색 `#888888` | ~3.9% |
| **활성(Full)** | 활성 플레이어 곡괭이 4개+ | 1.5 | 0.1 | 0.3 | 마젠타 `#FF00FF` | 2.8%~0.6% |

### 1-2 동적 목표 개수 (최대 4개)

| 활성 플레이어 곡괭이 수 | 시스템 곡괭이 목표 | 예상 HE 범위 |
|------------------------|------------------|--------------|
| 0~3개 (솔로) | 1 (휴식 모드) | ~53~55% |
| 4~20개 (~1~13명) | 1 (활성) | ~53~55% |
| 21~50개 (~14~33명) | 2 (활성) | ~52~54% |
| 51~80개 (~34~53명) | 3 (활성) | ~52~53% |
| 81~150개 (~54~100명) | 4 (활성) | ~52~53% |

### 1-3 앵커 곡괭이

- 항상 1개: `lifetime: Infinity` — 플레이어 0명일 때도 월드 유지
- 동적 추가분: `lifetime: 60000ms` (60초), 자연 소멸 후 타깃에 따라 재스폰
- 스폰 간격: 5000ms (현행 유지)

### 1-4 시각적 표현

- 휴식 모드: 회색/반투명, 작은 크기, 거의 움직이지 않음
- UI 툴팁: "PIKIT (Resting)" vs "PIKIT"
- 활성 모드: 현행 마젠타 그대로

## 변경 3: 곡괭이별 보상 분포 목표

곡괭이 1개 기준, 구매 가격 대비 회수율:

| 구간 | 확률 | 회수율 | 내용 |
|------|------|--------|------|
| 바닥 | 55% | 5~35% | common 블록만 파괴 (항상 보장) |
| 중간 | 25% | 35~70% | copper/iron 추가 |
| 근접 | 12% | 70~99% | emerald 히트 |
| **수익** | **8%** | **100%+** | gold/diamond 히트 → 투자금 초과 획득 |

- 평균 ROI ≈ 44.7% → HE ≈ 55.3% (단독 분포 기준)
- 어댑티브 시스템 + 잭팟 풀 조합으로 실제 HE 52~53%로 수렴
- **바닥 보장 원리**: common 블록(60% 가중치, HP 2~3)이 항상 존재해 0 회수 불가

## 변경 4: 잭팟 동적 풀 시스템

### 4-1 적립 메커니즘

```
곡괭이 소멸 이벤트 (_expirePickaxe):
  pickaxeProfit = pickaxe.price - pickaxe.totalReward
  if pickaxeProfit > 0:
    engine.houseProfitAccumulator += pickaxeProfit

  // 5만 크레딧 마일스톤마다 잭팟에 2,500 할당
  while engine.houseProfitAccumulator >= 50,000:
    engine.jackpotPool += 2,500
    engine.houseProfitAccumulator -= 50,000
```

- 하우스 수익의 5% → 잭팟 풀 (50K 수익당 2,500cr)
- 플레이어가 없거나 적을 때는 풀 적립 속도 느림 (자연스러운 억제)

### 4-2 스폰 조건

| 조건 | 기준값 |
|------|--------|
| 최소 풀 | **50,000 크레딧** |
| 스폰 확률 | 청크 블록 위치당 **0.01%** (현행 0.05% → 극악화) |
| 이미 존재 시 | 스폰 불가 |

### 4-3 HP 설계 (레이드)

- **HP: 1,500** (현행 300 → 5배)
- 10~15명 집중 공격 시 30~50초 소요
- TNT 저항 40% 유지

### 4-4 파괴 / 이탈 처리

| 이벤트 | 처리 |
|--------|------|
| 파괴 (마지막 타격자) | jackpotPool 전액 → 마지막 타격자 지급, pool = 0 |
| 화면 밖 이탈 | pool 유지 (계속 누적), HP 완전 회복, 재스폰 확률 0.001% |

### 4-5 알림 텍스트 변경

- 등장: `🏆 Jackpot Block! Pool: {N}cr 🏆`
- 파괴: `[이름] mined Jackpot! +{pool}cr`
