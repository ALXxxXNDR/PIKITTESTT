# PIKIT 기능명세서 — Epic 1: 게임 접속 및 관전

> 작성 기준: v4.7 | 작성일: 2026-03-19
> 관점: 유저 플로우 (비개발자용)

---

### [STORY 1-1] 유저는 처음 웹사이트에 접속하면 로그인 없이 게임 화면을 즉시 볼 수 있어야 한다.

1. 유저가 브라우저에서 PIKIT 주소를 입력해 접속한다.
2. 게임 화면(캔버스)이 로그인 과정 없이 즉시 표시된다.
3. 화면에는 곡괭이가 낙하하고 블록이 파괴되는 실시간 게임 장면이 나타난다.
4. 다른 플레이어들의 활동도 실시간으로 함께 표시된다.
5. 화면 상단에는 현재 채굴 깊이와 크레딧 잔액(0)이 표시된다.
6. 화면 하단에는 상점, 퀘스트, 랭킹 버튼이 표시된다.
7. 유저는 별도의 가입이나 로그인 없이 게임을 구경할 수 있는 상태가 된다.

---

### [STORY 1-2] 유저는 화면 상단에서 현재 게임의 실시간 정보를 한눈에 확인할 수 있어야 한다.

1. 유저가 게임 화면을 보고 있을 때 화면 상단 정보 표시줄이 항상 노출된다.
2. **채굴 깊이**: 현재 게임이 얼마나 깊이 진행되고 있는지 미터(m) 단위로 표시되며, 실시간으로 갱신된다.
3. **크레딧 잔액**: 화면 상단 우측에 현재 보유 크레딧이 숫자로 표시된다. 로그인 전에는 0으로 표시된다.
4. **플레이어 수 및 활성 곡괭이 수**: 메뉴 패널을 열면 현재 필드에 접속 중인 플레이어 수와 낙하 중인 곡괭이 총 개수를 확인할 수 있다.
5. 화면의 모든 수치는 게임 서버 상태에 따라 자동으로 실시간 갱신된다.

---

### [STORY 1-3] 유저는 메뉴 버튼을 클릭해 게임 정보와 블록 가이드 패널을 열 수 있어야 한다.

1. 유저가 화면 상단 좌측의 메뉴 버튼(햄버거 아이콘)을 클릭한다.
2. 화면 왼쪽에서 메뉴 패널이 슬라이드되어 나타난다. 동시에 패널 뒤쪽 배경이 반투명하게 어두워진다.
3. 패널에는 두 가지 정보 섹션이 표시된다:
   - **게임 현황**: 현재 접속 중인 플레이어 수와 낙하 중인 곡괭이 총 개수
   - **블록 가이드**: 게임에 등장하는 모든 블록의 종류, 획득 보상(크레딧), 등장 확률(%) 목록
4. 블록 가이드는 보상이 높은 블록 순서로 나열된다 (잭팟 블록 → 다이아몬드 블록 → 골드 블록 → ... → 흙 블록).
5. 각 블록 항목에는 블록 이미지, 이름, 보상 크레딧, 등장 확률이 함께 표시된다.
6. 유저가 패널 바깥의 어두운 영역을 클릭하면 패널이 닫힌다.
7. 유저가 패널 내의 X 버튼을 클릭해도 패널이 닫힌다.

---

### [STORY 1-4] 유저는 희귀 블록이 등장했을 때 화면 상단에 알림 배너를 볼 수 있어야 한다.

1. 게임 필드에 다이아몬드 블록 또는 잭팟 블록이 새로 등장한다.
2. 같은 필드에 접속 중인 모든 유저(관전자 포함)의 화면 최상단에 알림 배너가 즉시 표시된다.
3. **다이아몬드 블록 등장 시**: 배너에 "💎 Diamond Block appeared! (4,500 credits)" 문구가 표시된다.
4. **잭팟 블록 등장 시**: 배너에 "🏆 Jackpot Block appeared! (250,000 credits)" 문구가 표시되며, 일반 배너와 다른 특별한 색상 스타일로 강조된다.
5. 배너는 4초 후 자동으로 사라진다.

---

### [STORY 1-5] 유저는 다른 플레이어가 희귀 블록을 채굴했을 때 알림 배너를 볼 수 있어야 한다.

1. 다른 플레이어의 곡괭이가 골드 블록, 다이아몬드 블록, 또는 잭팟 블록을 파괴한다.
2. 같은 필드에 접속 중인 모든 유저의 화면에 채굴 알림 배너가 나타난다.
3. **골드 블록 채굴 시**: "✨ {플레이어 닉네임} mined Gold Block! +1,800" 문구가 담긴 배너가 10초 동안 표시된다.
4. **다이아몬드 블록 채굴 시**: "💎 {플레이어 닉네임} mined Diamond Block! +4,500" 문구가 담긴 배너가 10초 동안 표시된다.
5. **잭팟 블록 채굴 시**: "🏆 {플레이어 닉네임} mined Jackpot Block! +250,000" 문구가 담긴 배너가 15초 동안 표시된다.
6. 배너는 시간이 지나면 자동으로 페이드아웃되며 사라진다.

---

### [STORY 1-6] 유저는 잭팟 이벤트가 발생했을 때 별도 알림을 볼 수 있어야 한다.

1. 잭팟 블록 파괴 등 잭팟 이벤트 조건이 충족된다.
2. 같은 필드에 접속 중인 모든 유저의 화면에 "🎉 {플레이어 닉네임} found {보상 크레딧} from {블록 이름}!" 형식의 잭팟 전용 알림이 표시된다.
3. 알림은 4초 후 자동으로 사라진다.
4. **잭팟 블록 등장 조건 참고**: 게임 전체에서 누적 소비 크레딧이 150만 크레딧을 넘고, 동시 접속 플레이어가 10명 이상일 때 매우 낮은 확률(0.05%)로 등장한다.

---

### [STORY 1-7] 유저는 다른 플레이어가 곡괭이를 구매하면 그 곡괭이가 게임 화면에 즉시 등장하는 것을 볼 수 있어야 한다.

1. 다른 플레이어가 상점에서 곡괭이를 구매한다.
2. 관전 중인 유저의 화면에 새 곡괭이가 필드 상단에서 낙하를 시작하는 모습이 50밀리초(약 즉시) 이내에 렌더링된다.
3. 곡괭이는 8비트 픽셀 아트 스타일로 그려지며, 종류마다 외형이 다르다 (기본: 갈색, 파워: 청록색, 라이트: 금색, 스위프트: 은색).

---

### [STORY 1-8] 유저는 모바일 기기에서 접속해도 게임 화면이 정상적으로 표시되어야 한다.

1. 유저가 스마트폰 또는 태블릿 브라우저로 PIKIT에 접속한다.
2. 게임 화면이 기기 화면 크기에 맞춰 자동으로 조정되어 표시된다. 화면 확대나 축소는 자동으로 방지된다.
3. 화면 상단 정보 표시줄, 하단 버튼, 각종 팝업 및 패널이 모두 터치 화면에 적합하게 표시된다.
4. 메뉴 패널, 상점 패널 등은 버튼 탭으로 열고 닫을 수 있다.

---

### [STORY 1-9] 유저는 화면 하단의 채팅창에서 다른 플레이어들의 대화를 볼 수 있어야 한다.

1. 게임 화면 하단에 채팅 영역이 항상 표시된다.
2. 현재 필드에 있는 플레이어들의 채팅 메시지가 실시간으로 나타난다.
3. 플레이어 입장 및 퇴장 시 "{닉네임} joined the game." 또는 "{닉네임} left the game." 시스템 메시지가 채팅 영역에 별도 스타일로 표시된다.
4. 채팅창에는 최근 메시지 100개까지 유지되며, 초과 시 오래된 메시지부터 자동 삭제된다.
5. 채팅 버튼을 클릭하면 메시지 입력창이 열린다.
6. **비로그인 상태에서 채팅 전송을 시도하면** 로그인(지갑 연결) 안내 화면으로 자동 이동된다. 메시지는 전송되지 않는다.

---

### [STORY 1-10] 서버 연결이 끊겼을 때 게임 화면이 멈추는 상황을 유저가 경험할 수 있다. (자동 재연결 시도)

1. 유저가 게임을 관전하는 중 인터넷 연결이 끊어진다.
2. 게임 화면이 마지막으로 수신한 상태에서 정지된다(더 이상 움직이지 않음).
3. 서버와의 연결이 자동으로 재시도된다.
4. 재연결에 성공하면 게임 화면이 다시 실시간으로 갱신되기 시작한다.
5. 재연결에 최종 실패하면 화면은 정지 상태로 유지된다.

---

### [STORY 1-11] 재연결 성공 시 서버로부터 전체 게임 상태 스냅샷을 수신해야 한다.

1. WebSocket 재연결 성공 시 서버는 즉시 전체 게임 상태(블록 위치, 활성 곡괭이 목록, 잭팟 풀 금액, 현재 플레이어 목록)를 스냅샷으로 전송한다.
2. 클라이언트는 스냅샷을 수신하면 캔버스를 최신 상태로 완전 갱신한다.
3. 재연결 중 놓친 이벤트(배너 알림 등)는 복구하지 않으며, 재연결 이후 이벤트만 수신한다.

---

### [STORY 1-12] 연결이 끊어졌을 때 유저에게 재연결 시도 중임을 시각적으로 알려야 한다.

1. WebSocket 연결이 끊어지면 즉시 화면 상단 또는 중앙에 "연결 끊김 — 재연결 중..." 오버레이 또는 배너를 표시한다.
2. 게임 캔버스는 마지막 수신 프레임에서 정지된다.
3. 재연결에 성공하면 오버레이가 사라지고 게임이 재개된다.
4. 재연결에 영구 실패(3회 이상 시도 후)하면 "서버와의 연결이 끊겼습니다. 새로고침 해주세요." 메시지를 표시한다.

---

### [STORY 1-13] 최초 접속 시 서버가 응답하지 않으면 오류 화면을 표시해야 한다.

1. 유저가 처음 URL에 접속했을 때 서버가 응답하지 않거나 WebSocket 연결이 실패하면 빈 캔버스 대신 "서버에 연결할 수 없습니다. 잠시 후 다시 시도해 주세요." 오류 메시지를 표시한다.
2. 자동으로 5초마다 재시도하며, 재시도 중임을 카운트다운으로 안내한다.

---

### [STORY 1-14] 다수의 알림 배너가 동시에 발생하면 우선순위에 따라 순서대로 표시해야 한다.

1. 잭팟 스폰, 다이아몬드 파괴, 골드 파괴 배너가 동시에 발생하면 잭팟 > 다이아몬드 > 골드 순서로 우선순위를 적용한다.
2. 배너는 동시에 최대 2개까지 수직으로 쌓여 표시되며, 대기 중인 배너는 현재 배너가 사라진 후 순서대로 표시된다.
3. 같은 유형의 배너가 연속으로 발생하면 기존 배너를 갱신(타이머 초기화)한다.

---

### [STORY 1-15] 모바일에서 가로 모드로 전환하면 세로 모드를 사용하도록 안내해야 한다.

1. 모바일 기기에서 가로 모드(landscape)로 접속하거나 전환하면 "최적 화면 경험을 위해 세로 모드를 사용해 주세요." 안내 오버레이를 표시한다.
2. 오버레이는 세로 모드로 돌아올 때까지 유지된다. 게임은 오버레이 뒤에서 계속 실행된다.
