# PIKIT 기능명세서 — Epic 2: 로그인

> 작성 기준: v4.7 | 작성일: 2026-03-19
> 관점: 유저 플로우 (비개발자용)

---

### [STORY 2-1] 유저는 닉네임을 입력해 게임에 참여할 수 있어야 한다.

1. 관전 중인 유저가 상점, 퀘스트, 랭킹, 채팅 전송 등 로그인이 필요한 기능을 클릭한다.
2. 로그인 안내 화면(지갑 연결 팝업)이 자동으로 열린다.
3. 유저가 닉네임을 입력하고 참여 버튼을 클릭한다.
4. 닉네임이 처리되어 플레이어로 등록된다.
5. 초기 크레딧 10,000이 자동으로 지급된다.
6. 화면 상단 크레딧 잔액이 10,000으로 갱신된다.
7. 채팅창에 "{닉네임} joined the game." 시스템 메시지가 현재 필드 전체에 표시된다.
8. 유저는 이제 곡괭이 구매 등 게임에 직접 참여할 수 있는 상태가 된다.

---

### [STORY 2-2] 유저가 허용되지 않는 특수문자를 입력하면 자동으로 제거되어야 한다.

1. 유저가 닉네임 입력란에 `<`, `>`, `&`, `"`, `'`, `/` 등의 특수문자가 포함된 닉네임을 입력한다.
2. 게임 서버가 해당 특수문자들을 자동으로 제거한다.
3. 특수문자가 제거된 닉네임으로 플레이어가 정상 등록된다.
4. 유저는 정제된 닉네임으로 게임에 참여된 것을 확인한다.

---

### [STORY 2-3] 유저가 닉네임을 12자 초과로 입력하면 자동으로 잘려야 한다.

1. 유저가 13자 이상의 닉네임을 입력한다.
2. 서버가 앞에서부터 12자만 사용하고 나머지를 버린다.
3. 12자로 잘린 닉네임으로 플레이어가 정상 등록된다.

---

### [STORY 2-4] 유저가 공백만 입력하거나 아무것도 입력하지 않으면 기본 닉네임이 부여되어야 한다.

1. 유저가 닉네임 입력란을 비워 두거나 공백만 입력하고 참여를 시도한다.
2. 서버가 입력값이 실질적으로 비어 있음을 확인하고 자동으로 "Player"라는 기본 닉네임을 부여한다.
3. 유저는 "Player"라는 닉네임으로 정상 등록된다.

---

### [STORY 2-5] 유저는 MetaMask 지갑을 연결해 로그인할 수 있어야 한다.

1. 유저가 로그인이 필요한 기능(예: 상점 버튼)을 클릭한다.
2. 지갑 연결 안내 팝업이 열린다.
3. 유저가 팝업에서 MetaMask를 선택한다.
4. MetaMask 확장 프로그램이 지갑 연결 승인을 요청하는 팝업을 표시한다.
5. 유저가 연결을 승인한다.
6. 지갑 연결이 완료되면, 게임 로그인을 위한 서명 요청 팝업이 MetaMask에 표시된다.
7. 유저가 서명에 동의한다.
8. 서버가 서명을 검증하고 플레이어를 등록한다. 닉네임은 지갑 주소 축약형(예: 0x1234...abcd)으로 자동 설정된다.
9. 화면 상단 크레딧 잔액이 10,000으로 갱신된다.
10. 채팅창에 입장 메시지가 표시된다.
11. 유저는 지갑 연결 상태로 게임에 참여한다. (입출금, 퀘스트 등 지갑 전용 기능 사용 가능)

---

### [STORY 2-6] 유저가 MetaMask 서명 요청을 거부하면 로그인이 중단되고 관전 상태로 남아야 한다.

1. 지갑 연결은 완료되었지만 서명 요청 팝업이 MetaMask에 표시된 상태이다.
2. 유저가 MetaMask 팝업에서 "거부(Reject)" 버튼을 클릭한다.
3. 서명이 취소되어 게임 로그인이 진행되지 않는다.
4. 유저는 관전자 상태로 남는다. 화면 상단 크레딧 잔액은 0으로 유지된다.
5. 서명이 거부되었음을 안내하는 메시지가 표시될 수 있다.

---

### [STORY 2-7] 유저가 지갑 연결 팝업의 X 버튼을 클릭해 닫으면 로그인이 중단되어야 한다.

1. 유저가 상점 버튼 클릭 등으로 지갑 연결 팝업이 열린 상태이다.
2. 유저가 팝업의 X 버튼을 클릭해 닫는다.
3. 지갑이 연결되지 않은 채로 팝업이 닫힌다.
4. 유저는 관전자 상태로 유지된다.
5. 상점 패널 등 원래 시도했던 기능은 열리지 않는다. 별도의 오류 메시지 없이 원래 게임 화면으로 돌아온다.

---

### [STORY 2-8] 이미 로그인된 상태에서 다시 로그인을 시도하면 차단되어야 한다.

1. 유저가 이미 닉네임 또는 지갑으로 로그인되어 있는 상태이다.
2. 어떠한 경로로든 동일한 연결에서 로그인을 다시 시도한다.
3. 서버가 이미 로그인된 상태임을 확인하고 두 번째 시도를 무시한다.
4. 기존 로그인 상태가 그대로 유지된다. 중복 로그인으로 인한 데이터 손상은 발생하지 않는다.

---

### [STORY 2-9] 유저는 내 정보 팝업에서 로그아웃 버튼을 눌러 게임을 나갈 수 있어야 한다.

1. 로그인된 유저가 화면 상단 우측의 내 정보 버튼을 클릭한다.
2. 내 정보 팝업이 열린다.
3. 유저가 팝업 내 "Logout" 버튼을 클릭한다.
4. 브라우저에 "Are you sure you want to logout?" 확인 다이얼로그가 표시된다.
5. 유저가 "확인(OK)"을 클릭한다.
6. 로그아웃이 처리된다. 지갑 연결 상태였다면 지갑 연결도 함께 해제된다.
7. 페이지가 새로고침되어 유저는 다시 비로그인 관전자 상태로 돌아온다.

---

### [STORY 2-10] 유저가 로그아웃 확인 다이얼로그에서 취소를 클릭하면 로그아웃이 취소되어야 한다.

1. 내 정보 팝업에서 "Logout" 버튼을 클릭해 확인 다이얼로그가 표시된 상태이다.
2. 유저가 "취소(Cancel)"를 클릭한다.
3. 아무런 변화가 없다. 로그인 상태가 그대로 유지된다.
4. 내 정보 팝업은 열린 채로 유지된다.

---

### [STORY 2-11] 유저가 브라우저를 새로고침하면 세션이 종료되어야 한다.

1. 로그인 중인 유저가 브라우저의 새로고침 버튼을 누르거나 F5를 입력한다.
2. 기존 서버 연결이 끊어진다.
3. 서버는 해당 플레이어를 게임에서 제거하고, 같은 필드의 다른 유저들에게 "{닉네임} left the game." 메시지를 표시한다.
4. 게임 진행 데이터(크레딧, 활동 내역 등)는 저장되지 않고 초기화된다.
5. 페이지 재로드 후 유저는 비로그인 관전자 상태로 처음부터 다시 시작한다. 재로그인 시 크레딧 10,000이 새로 지급된다.

---

### [STORY 2-12] MetaMask가 설치되어 있지 않은 환경에서는 설치를 안내해야 한다.

1. 유저가 지갑 연결 버튼을 클릭했을 때 MetaMask가 설치되어 있지 않으면 "MetaMask가 설치되어 있지 않습니다." 메시지와 함께 MetaMask 공식 설치 링크 또는 닉네임 로그인으로 대신 시작하는 옵션을 제공한다.
2. 모바일 환경에서는 MetaMask 앱으로 연결하는 딥링크를 제공한다.

---

### [STORY 2-13] 동일한 지갑 주소로 여러 탭에서 동시 접속을 시도하면 기존 세션을 종료해야 한다.

1. 이미 로그인된 지갑 주소로 다른 탭 또는 기기에서 로그인을 시도하면 서버는 기존 WebSocket 세션을 강제 종료하고 새 세션을 활성화한다.
2. 기존 세션에는 "다른 곳에서 접속하여 연결이 종료되었습니다." 알림을 표시한다.
3. 이를 통해 동일 지갑의 크레딧 이중 지급을 방지한다.

---

### [STORY 2-14] 서명 메시지에는 재사용 방지를 위한 nonce와 타임스탬프가 포함되어야 한다.

1. 로그인 서명 요청 시 서버는 해당 세션에만 유효한 고유 nonce와 타임스탬프를 포함한 메시지를 생성한다.
2. 동일 서명을 가로채서 재사용하는 리플레이 공격을 방지하기 위해 서버는 각 nonce를 1회만 허용하고 만료 시간(5분) 이후의 서명을 거부한다.
3. 서명 검증 실패 시 로그인을 거부하고 "인증에 실패했습니다. 다시 시도해 주세요." 메시지를 표시한다.

---

### [STORY 2-15] 세션 중 MetaMask에서 지갑 연결이 해제되면 자동으로 로그아웃 처리해야 한다.

1. 게임 세션 중 유저가 MetaMask 확장에서 직접 계정 연결을 해제하거나 다른 계정으로 전환하면 클라이언트는 이를 감지하고 자동으로 로그아웃 처리한다.
2. "지갑 연결이 해제되어 로그아웃되었습니다." 메시지를 표시하고 관전 상태로 전환한다.
3. 진행 중이던 곡괭이는 서버에서 즉시 제거되며 만료 보상 알림은 표시되지 않는다.

---

### [STORY 2-16] 동일 닉네임이 이미 접속 중인 경우 번호를 붙여 구분해야 한다.

1. 닉네임 로그인 시 동일한 닉네임(예: "Player")이 이미 같은 필드에 접속 중이라면 서버는 중복을 피하기 위해 자동으로 번호를 붙인다(예: "Player#2", "Player#3").
2. 유저에게는 최종 할당된 닉네임을 알림으로 표시한다.
3. 이를 통해 채팅 및 알림에서 플레이어를 명확히 구분할 수 있다.
