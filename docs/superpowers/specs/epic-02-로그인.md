# [Epic 2] 로그인 서비스
> 작성 기준: v5.0 | 작성일: 2026-03-24
> 관점: 유저 플로우 (비개발자용)

## 목적
유저가 MetaMask 지갑 연결 또는 닉네임 입력으로 간편하게 로그인하여 게임에 참여할 수 있도록 한다.

---

### [STORY 2-1] 유저는 MetaMask 지갑을 연결해 로그인할 수 있어야 한다.
1. 관전 중인 유저가 로그인이 필요한 기능(상점, 퀘스트, 채팅 전송 등)을 클릭한다.
2. RainbowKit 지갑 연결 모달이 열린다.
3. 유저가 MetaMask를 선택하고 연결을 승인한다.
4. 게임 로그인을 위한 서명 요청이 MetaMask에 표시된다 (nonce + timestamp 포함, 5분 만료).
5. 유저가 서명에 동의하면 서버가 서명을 검증하고 플레이어를 등록한다.
6. 닉네임은 지갑 주소 축약형(예: 0x1234...abcd)으로 자동 설정된다.
7. 초기 크레딧 10,000이 자동으로 지급되고, 화면 상단 잔액이 갱신된다.
8. 채팅창에 "{닉네임} joined the game." 시스템 메시지가 표시된다.

---

### [STORY 2-2] 유저는 닉네임을 입력해 게임에 참여할 수 있어야 한다.
1. RainbowKit 지갑 연결 모달에서 닉네임 입력 옵션을 선택한다.
2. 유저가 닉네임을 입력하고 참여 버튼을 클릭한다.
3. 닉네임이 처리되어 플레이어로 등록된다.
4. 초기 크레딧 10,000이 자동으로 지급되고, 화면 상단 잔액이 갱신된다.
5. 채팅창에 "{닉네임} joined the game." 시스템 메시지가 표시된다.

---

### [STORY 2-3] 닉네임은 12자 제한이며, 특수문자는 자동 제거되어야 한다.
1. 유저가 닉네임을 입력하면 `<`, `>`, `&`, `"`, `'`, `/` 등의 특수문자가 자동으로 제거된다.
2. 13자 이상 입력 시 앞에서부터 12자만 사용하고 나머지는 버린다.
3. 공백만 입력하거나 아무것도 입력하지 않으면 "Player"라는 기본 닉네임이 부여된다.

---

### [STORY 2-4] 동일 닉네임이 이미 접속 중이면 번호가 자동 부여되어야 한다.
1. 닉네임 로그인 시 동일한 닉네임이 이미 접속 중이면 자동으로 번호를 붙인다 (예: "Player#2", "Player#3").
2. 유저에게 최종 할당된 닉네임을 알림으로 표시한다.

---

### [STORY 2-5] 유저가 서명을 거부하거나 팝업을 닫으면 로그인이 중단되어야 한다.
1. MetaMask 서명 요청을 거부하면 로그인이 진행되지 않고 관전자 상태로 남는다.
2. 지갑 연결 모달의 X 버튼을 클릭해 닫으면 관전자 상태가 유지된다.
3. 별도의 오류 메시지 없이 원래 게임 화면으로 돌아온다.

---

### [STORY 2-6] 유저는 내 정보 팝업에서 로그아웃할 수 있어야 한다.
1. 로그인된 유저가 화면 상단 우측의 내 정보 버튼을 클릭한다.
2. 내 정보 팝업에서 "Logout" 버튼을 클릭한다.
3. "Are you sure you want to logout?" 확인 다이얼로그가 표시된다.
4. "확인(OK)" 클릭 시 로그아웃이 처리된다. 지갑 연결 상태였다면 지갑 연결도 함께 해제된다.
5. 페이지가 새로고침되어 비로그인 관전자 상태로 돌아온다.
6. "취소(Cancel)" 클릭 시 아무 변화 없이 팝업이 유지된다.

---

### [STORY 2-7] 브라우저 새로고침 시 세션이 종료되어야 한다.
1. 로그인 중인 유저가 브라우저를 새로고침하면 기존 서버 연결이 끊어진다.
2. 서버는 해당 플레이어를 제거하고, 같은 필드의 유저들에게 "{닉네임} left the game." 메시지를 표시한다.
3. 페이지 재로드 후 유저는 비로그인 관전자 상태로 처음부터 시작한다.

---

### [STORY 2-8] 동일 지갑으로 중복 접속 시 기존 세션이 강제 종료되어야 한다.
1. 이미 로그인된 지갑 주소로 다른 탭 또는 기기에서 로그인을 시도하면, 서버는 기존 세션을 강제 종료하고 새 세션을 활성화한다.
2. 기존 세션에 "다른 곳에서 접속하여 연결이 종료되었습니다." 알림을 표시한다.

---

### [STORY 2-9] MetaMask 연결 해제 시 자동 로그아웃되어야 한다.
1. 게임 세션 중 유저가 MetaMask에서 직접 계정 연결을 해제하거나 다른 계정으로 전환하면 자동으로 로그아웃 처리된다.
2. "지갑 연결이 해제되어 로그아웃되었습니다." 메시지를 표시하고 관전 상태로 전환한다.
3. 진행 중이던 곡괭이는 즉시 제거된다.

---

### [STORY 2-10] 서명 메시지에는 재사용 방지를 위한 nonce와 타임스탬프가 포함되어야 한다.
1. 로그인 서명 요청 시 해당 세션에만 유효한 고유 nonce와 타임스탬프를 포함한다.
2. 각 nonce는 1회만 허용되며, 5분 만료 이후의 서명은 거부된다.
3. 서명 검증 실패 시 "인증에 실패했습니다. 다시 시도해 주세요." 메시지를 표시한다.

---

## 정책 요약

| 항목 | 내용 |

| 로그인 방법 | MetaMask 지갑 연결 (RainbowKit) 또는 닉네임 입력 |
| 초기 크레딧 | 10,000 자동 지급 |
| 닉네임 제한 | 12자, 특수문자 자동 제거, 공백 시 "Player" |

| 서명 만료 | nonce + timestamp, 5분 |

| 중복 접속 (지갑) | 기존 세션 강제 종료 |
| MetaMask 연결 해제 | 자동 로그아웃 |
