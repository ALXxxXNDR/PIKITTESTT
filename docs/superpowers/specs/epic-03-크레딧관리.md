# [Epic 3] 크레딧 관리 서비스
> 작성 기준: v5.0 | 작성일: 2026-03-24
> 관점: 유저 플로우 (비개발자용)

## 목적
유저의 크레딧 잔액을 실시간으로 관리하고, USDC 입출금을 통해 충전 크레딧과 인게임 크레딧을 구분하여 안전하게 운영한다.

---

### [STORY 3-1] 유저는 로그인 시 초기 크레딧 10,000을 자동으로 받아야 한다.
1. 유저가 닉네임 입력 또는 지갑 연결로 로그인에 성공한다.
2. 별도의 버튼 클릭 없이 즉시 10,000 크레딧이 자동 지급된다.
3. 화면 상단 잔액이 10,000으로 표시된다.
4. 유저는 바로 곡괭이 구매 등 게임 활동을 시작할 수 있다.

---

### [STORY 3-2] 크레딧은 충전 크레딧과 인게임 크레딧으로 구분되어야 한다.
1. **충전 크레딧 (출금 가능)**: USDC 입금, 블록 채굴 보상으로 획득한 크레딧.
2. **인게임 크레딧 (출금 불가)**: 퀘스트 보상으로 획득한 크레딧.
3. 화면 상단 잔액에 마우스를 올리거나 탭하면 충전 크레딧과 인게임 크레딧을 구분해 보여주는 툴팁이 나타난다.

---

### [STORY 3-3] 유저는 화면 상단에서 크레딧 잔액을 실시간으로 확인할 수 있어야 한다.
1. 로그인 후 화면 상단 우측에 현재 크레딧 잔액이 항상 표시된다.
2. 잔액은 블록 파괴 보상, 곡괭이 구매, 입금/출금 등 변화가 생길 때마다 자동 갱신된다.

---

### [STORY 3-4] 유저는 곡괭이 구매 후 잔액이 즉시 차감되는 것을 확인할 수 있어야 한다.
1. 유저가 상점에서 곡괭이를 구매한다.
2. 서버가 구매를 처리한 후 차감된 잔액이 즉시 반영된다.
3. 화면 상단 잔액이 구매 가격만큼 줄어든 새 금액으로 갱신된다.

---

### [STORY 3-5] 유저는 블록 파괴 보상을 받을 때 잔액이 즉시 증가해야 한다.
1. 유저의 곡괭이가 블록을 파괴한다.
2. 해당 블록의 보상 크레딧이 자동으로 잔액에 추가된다.
3. 화면 상단 잔액이 보상만큼 늘어난 새 금액으로 실시간 갱신된다.

---

### [STORY 3-6] 잔액이 부족하면 구매가 불가능해야 한다.
1. 유저의 크레딧 잔액이 선택한 곡괭이 가격보다 부족한 상태이다.
2. 유저가 상점에서 해당 곡괭이를 클릭하면 서버가 구매를 거부한다.
3. 화면에 구매 불가 오류 메시지가 표시된다.

---

### [STORY 3-7] 유저는 내 정보에서 총 획득/소비/손익을 확인할 수 있어야 한다.
1. 유저가 내 정보 팝업의 개요 탭을 연다.
2. 다음 크레딧 통계를 확인할 수 있다:
   - **총 잔액**: 현재 보유 크레딧 전체
   - **충전 크레딧**: 입금, 블록 채굴 보상으로 받은 크레딧 (출금 가능)
   - **인게임 크레딧**: 퀘스트 보상으로 얻은 크레딧 (출금 불가)
   - **총 획득(Total Earned)**: 누적 획득 크레딧
   - **총 소비(Total Spent)**: 누적 소비 크레딧
   - **손익(Profit/Loss)**: 총 획득 - 총 소비 (이익 초록, 손실 빨강)
3. 모든 수치는 게임 활동에 따라 자동 갱신된다.

---

### [STORY 3-8] 유저는 USDC를 입금해 충전 크레딧을 추가할 수 있어야 한다. (지갑 연결 필수)
1. 유저가 내 정보 팝업에서 "Deposit" 탭을 클릭한다.
2. 입금할 USDC 금액을 입력한다. 1 USDC = 10,000 크레딧 비율로 미리보기가 자동 갱신된다.
3. "Deposit" 버튼을 클릭하면 MetaMask에서 USDC 사용 승인 요청이 표시된다.
4. 승인 후 입금 트랜잭션이 전송되고, "Deposit tx sent!" 알림이 표시된다.
5. 서버가 온체인 거래를 확인(2블록 이상)한 후 크레딧을 지급한다.
6. 트랜잭션이 실패(revert)하면 크레딧을 지급하지 않고 실패 알림을 표시한다.

---

### [STORY 3-9] 유저는 충전 크레딧을 USDC로 출금할 수 있어야 한다. (지갑 연결 필수)
1. 유저가 내 정보 팝업에서 "Withdraw" 탭을 클릭한다.
2. 출금할 USDC 금액을 입력한다. 1 USDC 출금 시 10,500 크레딧 소모 (5% 수수료 포함).
3. 보유 충전 크레딧이 부족하면 오류 메시지가 표시되고 출금이 중단된다.
4. 충전 크레딧이 충분하면 MetaMask 서명 요청이 표시된다.
5. 승인 후 출금 트랜잭션이 전송되고, 서버가 온체인 확인 후 크레딧을 차감한다.
6. 트랜잭션 실패 시 예약된 크레딧이 즉시 복구되고 복구 알림을 표시한다.
7. **인게임 크레딧(퀘스트 보상)은 출금 대상이 아니다.** 충전 크레딧만 출금 가능하다.

---

### [STORY 3-10] 구매 처리는 원자적으로 실행되어야 한다.
1. 서버는 크레딧 차감과 아이템 생성을 하나의 원자적 트랜잭션으로 처리한다.
2. 크레딧 차감 후 아이템 생성에 실패하면 차감된 크레딧을 즉시 복구한다.
3. 동시에 여러 구매 요청이 도착해도 크레딧 잔액이 절대 음수가 되어서는 안 된다.

---

## 정책 요약

| 항목 | 내용 |
|------|------|
| 초기 크레딧 | 로그인 시 10,000 자동 지급 |
| 충전 크레딧 (출금 가능) | USDC 입금, 블록 채굴 보상 |
| 인게임 크레딧 (출금 불가) | 퀘스트 보상 |
| USDC 입금 비율 | 1 USDC = 10,000 크레딧 |
| USDC 출금 비율 | 10,500 크레딧 = 1 USDC (5% 수수료) |
| 온체인 확인 | 2블록 이상 확인 후 크레딧 지급 |
| 잔액 보호 | 원자적 처리, 음수 방지 |
