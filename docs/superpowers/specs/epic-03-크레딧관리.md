# PIKIT 기능명세서 — Epic 3: 크레딧 관리

> 작성 기준: v4.7 | 작성일: 2026-03-19
> 관점: 유저 플로우 (비개발자용)

---

### [STORY 3-1] 유저는 로그인 시 초기 크레딧 10,000을 자동으로 받아야 한다.

1. 유저가 닉네임 입력 또는 지갑 연결로 로그인에 성공한다.
2. 별도의 버튼 클릭이나 추가 행동 없이 즉시 10,000 크레딧이 자동으로 지급된다.
3. 화면 상단 크레딧 잔액이 10,000으로 표시된다.
4. 유저는 바로 곡괭이 구매 등 게임 활동을 시작할 수 있다.

---

### [STORY 3-2] 유저는 무료 크레딧 충전 버튼으로 10,000 크레딧을 추가로 받을 수 있어야 한다.

1. 유저가 상점 패널을 열면 상단에 "+ 10,000 Credits" 버튼이 표시된다.
2. 또는 내 정보 팝업의 개요 탭에서 "+ 10,000 Credits (Free)" 버튼을 확인할 수 있다.
3. 유저가 해당 버튼을 클릭한다.
4. 10,000 크레딧이 즉시 크레딧 잔액에 추가된다.
5. 화면 상단 크레딧 잔액이 갱신된다.
6. 화면에 "10,000 credits added!" 성공 알림 메시지가 약 3초간 표시된다.

---

### [STORY 3-3] 유저가 무료 크레딧 충전을 30초 내에 5회 초과로 시도하면 추가 충전이 거부되어야 한다.

1. 유저가 30초 이내에 무료 충전 버튼을 6번 이상 클릭한다.
2. 6번째 시도부터 서버가 추가 충전 요청을 거부한다.
3. 크레딧 잔액이 변경되지 않는다.
4. 30초가 경과하면 충전 횟수 제한이 초기화되어 다시 5회 충전이 가능하다.

---

### [STORY 3-4] 유저는 화면 상단에서 크레딧 잔액을 실시간으로 확인할 수 있어야 한다.

1. 로그인 후 화면 상단 우측에 현재 크레딧 잔액이 항상 숫자로 표시된다.
2. 잔액은 블록 파괴 보상 획득, 곡괭이 구매, 무료 충전, 입금 등 변화가 생길 때마다 자동으로 갱신된다.
3. 유저가 화면 상단의 잔액 표시 영역에 마우스를 올리거나(모바일에서는 탭하면) 크레딧 상세 툴팁이 나타나며, **충전 크레딧**과 **인게임 크레딧**을 구분해 보여준다.
   - **충전 크레딧**: 초기 지급, 무료 충전, USDC 입금, 블록 채굴 보상으로 받은 크레딧 (출금 가능)
   - **인게임 크레딧**: 퀘스트 보상으로 획득한 크레딧 (출금 불가)
4. 내 정보 팝업 개요 탭에서도 잔액 전체 내역(총 획득, 총 소비, 손익)을 확인할 수 있다.

---

### [STORY 3-5] 유저는 곡괭이 구매 후 잔액이 즉시 차감되는 것을 확인할 수 있어야 한다.

1. 유저가 상점 패널에서 곡괭이를 선택해 구매한다.
2. 서버가 구매를 처리한 후 차감된 잔액 정보를 즉시 반환한다.
3. 화면 상단 크레딧 잔액이 구매 가격만큼 줄어든 새 금액으로 갱신된다.
4. 구매한 곡괭이가 게임 화면에 낙하하기 시작한다.

---

### [STORY 3-6] 유저는 블록 파괴 보상을 받을 때 잔액이 자동으로 증가하는 것을 확인할 수 있어야 한다.

1. 유저의 곡괭이가 블록을 파괴한다.
2. 서버가 해당 블록의 보상 크레딧을 자동으로 계산해 유저 잔액에 추가한다.
3. 화면 상단 크레딧 잔액이 보상 크레딧만큼 늘어난 새 금액으로 실시간 갱신된다.
4. 연속으로 블록을 빠르게 파괴하면 콤보 보너스가 적용되어 보상이 더 커진다. (2초 내 연속 3회 이상 파괴 시 최대 1.5배)
5. 희귀 블록 파괴 시 보상 획득 외에 채굴 알림 배너도 화면에 함께 표시된다.

---

### [STORY 3-7] 유저의 크레딧이 구매하려는 곡괭이 가격보다 부족하면 구매가 불가능해야 한다.

1. 유저의 크레딧 잔액이 선택한 곡괭이 가격보다 부족한 상태이다.
2. 유저가 상점에서 해당 곡괭이를 클릭한다.
3. 서버가 잔액 부족을 확인하고 구매를 거부한다.
4. 화면에 구매 불가 오류 메시지가 표시된다.
5. 유저는 무료 충전 버튼으로 크레딧을 보충하거나, USDC 입금을 통해 크레딧을 추가한 후 재시도할 수 있다.

---

### [STORY 3-8] 유저는 내 정보 팝업에서 총 획득 크레딧, 총 소비 크레딧, 손익을 확인할 수 있어야 한다.

1. 유저가 화면 상단 우측의 내 정보 버튼을 클릭해 팝업을 연다.
2. 개요 탭에서 다음 크레딧 통계를 확인할 수 있다:
   - **총 잔액**: 현재 보유 크레딧 전체
   - **충전 크레딧**: 입금, 무료 충전, 블록 채굴 보상으로 받은 크레딧 (출금 가능)
   - **인게임 크레딧**: 퀘스트 보상으로 얻은 크레딧 (출금 불가)
   - **총 획득(Total Earned)**: 지금까지 채굴 보상 등으로 획득한 누적 크레딧
   - **총 소비(Total Spent)**: 지금까지 곡괭이, TNT 구매 등에 사용한 누적 크레딧
   - **손익(Profit/Loss)**: 총 획득 - 총 소비. 이익이면 초록색, 손실이면 빨간색으로 표시된다.
3. 모든 수치는 게임 활동에 따라 자동으로 갱신된다.

---

### [STORY 3-9] 유저는 USDC를 입금해 충전 크레딧을 추가할 수 있어야 한다. (지갑 연결 필수)

1. 유저가 내 정보 팝업에서 "Deposit" 탭을 클릭한다.
2. 입금할 USDC 금액을 입력한다. 입력할 때마다 받게 될 크레딧 수량 미리보기가 자동 갱신된다. (1 USDC = 크레딧 10,000)
3. +/- 버튼으로 금액을 늘리거나 줄일 수 있다.
4. "Deposit" 버튼을 클릭한다.
5. MetaMask에서 USDC 사용 승인을 요청하는 팝업이 표시된다. 유저가 승인한다.
6. 실제 입금 트랜잭션이 자동으로 전송된다.
7. 화면에 "Deposit tx sent! X USDC → Y credits" 알림 메시지가 표시된다.
8. 서버가 온체인 거래를 확인하고 크레딧을 지급한다.
9. 크레딧 잔액이 갱신된다.

---

### [STORY 3-10] 유저는 충전 크레딧을 USDC로 출금할 수 있어야 한다. (지갑 연결 필수)

1. 유저가 내 정보 팝업에서 "Withdraw" 탭을 클릭한다.
2. 출금할 USDC 금액을 입력한다. 입력할 때마다 소모될 크레딧 수량 미리보기가 자동 갱신된다. (1 USDC 출금 시 크레딧 10,500 소모 — 5% 수수료 포함)
3. "Withdraw" 버튼을 클릭한다.
4. 보유 충전 크레딧이 부족한 경우 "Not enough credits. Need {필요 크레딧수}" 오류 메시지가 표시되고 출금이 중단된다.
5. 충전 크레딧이 충분한 경우 MetaMask에서 출금 트랜잭션 서명을 요청하는 팝업이 표시된다. 유저가 승인한다.
6. 화면에 "Withdraw tx sent! {크레딧} credits → {USDC} USDC" 알림 메시지가 표시된다.
7. 서버가 온체인 거래를 확인하고 크레딧을 차감한다.
8. 크레딧 잔액이 갱신된다.
9. **퀘스트 보상으로 얻은 인게임 크레딧은 출금 대상이 아니다.** 충전 크레딧(입금, 무료 충전, 블록 채굴 보상으로 받은 크레딧)만 출금할 수 있다.
