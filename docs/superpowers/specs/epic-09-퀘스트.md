## ✦ [EPIC 9] 퀘스트 서비스

### 목적
게임 내 활동(블록 파괴, 곡괭이 구매, 매일 로그인 등)으로 장기 목표를 달성하고, 블록체인에 완료를 등록하면 게임 내 크레딧 보상을 받는 시스템이다. 각 퀘스트는 여러 단계(Tier)로 구성되며, 이전 단계를 완료해야 다음 단계로 진행할 수 있다.

### [STORY 9-1] 유저는 퀘스트 버튼을 클릭해 퀘스트 목록 패널을 열 수 있어야 한다.

1. 유저가 화면의 "Quest" 버튼을 클릭한다.
2. 로그인 상태가 아닌 경우, 퀘스트 패널이 열리지 않고 로그인 안내가 표시된다. (Story 10-3 참고)
3. 로그인 상태인 경우, 퀘스트 목록 패널이 화면 옆에서 슬라이드 인 되며 열린다.
4. 패널 뒤로 반투명한 어두운 배경이 깔린다.
5. 서버에서 현재 퀘스트 진행 데이터를 불러와 목록이 화면에 표시된다.

---

### [STORY 9-2] 유저는 퀘스트 패널을 닫을 수 있어야 한다.

1. 방법 A: 퀘스트 패널 우측 상단의 X 버튼을 클릭한다.
2. 방법 B: 패널 외부의 어두운 반투명 배경 영역을 클릭한다.
3. 두 방법 모두 패널이 닫힌다.
4. 다른 패널(상점 등)이 열려 있지 않으면 어두운 배경도 함께 사라진다.

---

### [STORY 9-3] 로그인하지 않은 유저가 퀘스트 버튼을 클릭하면 로그인 안내가 표시되어야 한다.

1. 유저가 로그인하지 않은 상태에서 "Quest" 버튼을 클릭한다.
2. 퀘스트 패널은 열리지 않는다.
3. 지갑이 이미 연결된 경우: "로그인 중..." 안내 메시지가 잠깐 표시된 뒤 자동 로그인이 시도된다.
4. 지갑이 연결되지 않은 경우: 지갑 연결 모달이 열려 연결을 유도한다.

---

### [STORY 9-4] 유저는 각 퀘스트 항목에서 진행 상황을 확인할 수 있어야 한다.

1. 퀘스트 패널이 열리면 퀘스트 목록이 표시된다.
2. 각 퀘스트 항목에는 다음이 표시된다:
   - 퀘스트 이름과 설명
   - 달성 목표 수치 (예: "200개 파괴")
   - 현재까지 달성한 수치 (예: "47개")
   - 0%부터 100%까지의 진행률 막대
   - 완료 시 받을 보상 크레딧 수량
3. 목표를 이미 달성하고 온체인 완료까지 마친 퀘스트는 목록 하단에 "완료" 배지와 함께 표시된다.
4. 아직 완료되지 않은 퀘스트(진행 중 또는 완료 버튼 대기)는 목록 상단에 표시된다.

---

### [STORY 9-5] 유저가 게임에서 블록을 파괴하면 관련 퀘스트의 진행 카운터가 자동으로 증가해야 한다.

1. 유저의 곡괭이가 블록을 파괴한다.
2. 파괴한 블록 종류에 따라 해당 퀘스트의 진행 수치가 자동으로 1 증가한다.
   - 예: Dirt 블록 파괴 → "흙 채굴자" 퀘스트 카운터 +1
   - 모든 블록 파괴 → "블록 파괴자" 퀘스트 카운터 +1
3. 별도의 조작 없이 게임 플레이 중 자동으로 집계된다.
4. 다음 번 퀘스트 패널을 열면 업데이트된 수치를 확인할 수 있다.

---

### [STORY 9-6] 유저가 퀘스트 목표를 달성하면 "블록체인에 완료 등록" 버튼이 활성화되어야 한다.

1. 유저의 진행 수치가 목표값에 도달한다.
   - 예: Dirt 블록 100개 파괴 목표 → 100개 달성
2. 퀘스트 패널에서 해당 퀘스트 항목 하단에 "블록체인에 완료 등록" 버튼이 표시된다.
3. 이 버튼이 표시되기 전까지는 완료 처리를 진행할 수 없다.

---

### [STORY 9-7] 유저는 "완료 등록" 버튼을 클릭해 지갑 서명 요청을 받아야 한다.

1. 유저가 "블록체인에 완료 등록" 버튼을 클릭한다.
2. 지갑이 연결되지 않은 경우: "지갑을 먼저 연결해 주세요" 안내 메시지가 표시되고 처리가 중단된다. (Story 10-11 참고)
3. 지갑이 연결된 경우: 블록체인에서 이미 완료된 퀘스트인지 사전 확인한다.
4. 이미 블록체인에 등록되어 있다면 서버와 자동으로 동기화 처리된다.
5. 미완료 상태라면, 연결된 지갑 앱(예: 메타마스크)에 서명 요청 팝업이 나타난다.
6. 버튼 텍스트가 "전송 중..."으로 바뀌어 중복 클릭이 방지된다.

---

### [STORY 9-8] 유저가 서명에 동의하면 처리 중 상태가 표시되어야 한다.

1. 유저가 지갑 앱의 서명 요청 팝업에서 승인 버튼을 누른다.
2. 버튼 텍스트가 "확인 중..."으로 변경된다.
3. 화면에 "트랜잭션이 전송되었습니다. 확인 대기 중..." 안내 메시지가 표시된다.
4. 서버가 블록체인에서 트랜잭션이 정상 처리되었는지 확인하는 동안 대기 상태가 유지된다.

---

### [STORY 9-9] 블록체인 확인이 완료되면 퀘스트가 완료 처리되고 보상이 지급되어야 한다.

1. 서버가 블록체인에서 트랜잭션 성공을 확인한다.
2. 보내는 사람이 유저 본인의 지갑 주소와 일치하는지 검증된다.
3. 게임 내 목표 달성 여부가 재확인된다.
4. 모든 확인을 통과하면:
   - 해당 퀘스트 항목에 "✔ 완료" 배지가 표시된다.
   - 퀘스트 보상 크레딧이 즉시 게임 내 잔액에 적립된다.
   - 잔액 갱신이 화면에 반영된다.
5. 완료된 퀘스트는 목록 하단으로 이동한다.

---

### [STORY 9-10] 유저가 서명을 거부하면 오류 처리가 되어야 한다.

1. 유저가 지갑 앱의 서명 요청 팝업에서 "거부" 또는 "취소"를 누른다.
2. 오류 안내 메시지가 화면에 표시된다. (예: "사용자가 요청을 거부했습니다")
3. "블록체인에 완료 등록" 버튼이 원래 상태로 되돌아온다.
4. 퀘스트 완료 처리는 이루어지지 않으며 퀘스트 상태에 변경이 없다.

---

### [STORY 9-11] 지갑을 연결하지 않은 상태에서 완료 버튼을 클릭하면 연결 안내가 표시되어야 한다.

1. 유저가 지갑 미연결 상태에서 "블록체인에 완료 등록" 버튼을 클릭한다.
2. "지갑을 먼저 연결해 주세요" 오류 메시지가 표시된다.
3. 버튼 상태는 변하지 않는다.
4. 더 이상의 처리가 진행되지 않는다.

---

### [STORY 9-12] 이미 완료한 퀘스트를 다시 완료하려 하면 서버에서 거부되어야 한다.

1. 유저가 이미 완료 처리된 퀘스트의 완료 등록을 다시 시도한다.
2. 서버가 이미 완료된 퀘스트임을 확인하고 처리를 거부한다.
3. "이미 완료된 퀘스트입니다" 오류 메시지가 화면에 표시된다.
4. 보상이 중복으로 지급되지 않는다.

---

### [STORY 9-13] 잘못된 네트워크(Sepolia 외)에서 트랜잭션을 보내면 실패 처리가 되어야 한다.

1. 유저가 Sepolia 테스트넷이 아닌 다른 네트워크(예: 이더리움 메인넷)에서 트랜잭션을 시도한다.
2. 트랜잭션 전송 전 또는 전송 후 검증 단계에서 오류가 감지된다.
3. 서버가 해당 트랜잭션을 찾거나 인정하지 않아 실패 처리된다.
4. 오류 메시지가 화면에 표시된다.
5. "블록체인에 완료 등록" 버튼이 원래 상태로 복원된다.

---

### [STORY 9-14] 유저는 매일 로그인해 일일 플레이어 퀘스트를 진행할 수 있어야 한다.

1. 유저가 게임에 로그인한다.
2. 오늘 날짜로 처음 로그인하는 경우, 일일 플레이어 퀘스트의 로그인 일수 카운터가 자동으로 1 증가한다.
3. 같은 날 이미 로그인한 적이 있는 경우, 재접속해도 카운터는 증가하지 않는다.
4. 1일 로그인 목표 달성 시 "블록체인에 완료 등록" 버튼이 활성화된다.
5. 온체인 완료 처리를 마치면 1,000 크레딧 보상이 지급된다.
6. 상위 단계(7일, 30일, 100일, 365일 등)도 동일한 방식으로 진행되며, 단계가 높을수록 보상이 커진다.
