## ✦ [EPIC 12] 채팅 서비스

### 목적
같은 필드에 있는 플레이어들이 실시간으로 메시지를 주고받을 수 있는 서비스다.

---

### [STORY 12-1] 유저는 채팅 버튼을 클릭해 채팅 입력창을 열 수 있어야 한다.

1. 유저가 화면의 채팅 버튼을 클릭한다.
2. 채팅 입력창이 화면에 나타나고, 커서가 입력창 안으로 자동으로 이동한다.
3. 이전에 다른 플레이어들과 주고받은 대화 내역이 함께 표시된다.

---

### [STORY 12-2] 유저는 메시지를 입력한 후 Enter 키를 눌러 전송할 수 있어야 한다.

1. 유저가 채팅 입력창에 메시지를 입력한다.
2. Enter 키를 누른다.
3. 로그인 상태가 아닌 경우, 메시지가 전송되지 않고 로그인 안내가 표시된다. (Story 12-6 참고)
4. 로그인 상태인 경우, 메시지가 같은 필드의 모든 플레이어에게 전송된다.
5. 메시지 전송 후 입력창이 자동으로 비워진다.

---

### [STORY 12-3] 유저는 전송 버튼을 클릭해 메시지를 전송할 수 있어야 한다.

1. 유저가 채팅 입력창에 메시지를 입력한다.
2. 입력창 옆의 전송 버튼을 클릭한다.
3. Enter 키 전송과 동일하게 처리된다.
4. 메시지 전송 후 입력창이 자동으로 비워진다.

---

### [STORY 12-4] 유저는 Escape 키를 눌러 채팅 입력창을 닫을 수 있어야 한다.

1. 채팅 입력창이 열려 있는 상태에서 키보드의 Escape 키를 누른다.
2. 채팅 입력창이 닫힌다.
3. 입력 중이던 텍스트는 전송되지 않는다.

---

### [STORY 12-5] 유저는 채팅 버튼을 다시 클릭해 입력창을 닫을 수 있어야 한다.

1. 채팅 입력창이 열린 상태에서 채팅 버튼을 다시 클릭한다.
2. 채팅 입력창이 닫힌다.

---

### [STORY 12-6] 로그인하지 않은 유저가 채팅 전송을 시도하면 로그인 안내가 표시되어야 한다.

1. 유저가 로그인하지 않은 상태에서 Enter 키를 누르거나 전송 버튼을 클릭한다.
2. 메시지가 전송되지 않는다.
3. 지갑이 이미 연결된 경우: "로그인 중..." 안내 메시지가 잠깐 표시된 뒤 자동 로그인이 시도된다.
4. 지갑이 연결되지 않은 경우: 지갑 연결 모달이 열려 연결을 유도한다.
5. 참고: 채팅 버튼을 클릭해 입력창을 열고 이전 메시지를 보는 것은 로그인 없이도 가능하다. 메시지 전송만 로그인이 필요하다.

---

### [STORY 12-7] 유저가 200자를 초과하는 메시지를 입력하면 200자까지만 전송되어야 한다.

1. 유저가 200자보다 긴 텍스트를 입력하고 전송을 시도한다.
2. 서버에서 200자까지만 수신하여 처리한다.
3. 201번째 글자 이후의 내용은 무시되어 다른 플레이어에게 전달되지 않는다.

---

### [STORY 12-8] 유저가 1초 내에 연속으로 메시지를 전송하려 하면 두 번째 메시지는 무시되어야 한다.

1. 유저가 첫 번째 메시지를 전송한다.
2. 1초가 지나기 전에 두 번째 메시지를 전송하려 한다.
3. 두 번째 메시지는 서버에서 무시되어 다른 플레이어에게 전달되지 않는다.
4. 별도의 오류 메시지는 표시되지 않으며, 입력창은 정상적으로 비워진다.

---

### [STORY 12-9] 유저는 같은 필드의 다른 플레이어가 보낸 메시지를 실시간으로 볼 수 있어야 한다.

1. 같은 필드에 있는 다른 플레이어가 메시지를 전송한다.
2. 유저의 채팅창에 해당 메시지가 즉시 표시된다.
3. 메시지는 "[플레이어 이름] [메시지 내용]" 형태로 표시된다.
4. 새 메시지가 들어오면 채팅창이 자동으로 최신 메시지 위치로 스크롤된다.

---

### [STORY 12-10] 유저는 다른 플레이어가 현재 필드에 입장했을 때 시스템 메시지를 확인할 수 있어야 한다.

1. 다른 플레이어가 현재 필드에 접속한다.
2. 채팅창에 "[플레이어명] joined" 형태의 시스템 메시지가 표시된다.
3. 시스템 메시지는 일반 채팅 메시지와 다른 색상(회색 이탤릭체)으로 구분된다.

---

### [STORY 12-11] 유저는 다른 플레이어가 현재 필드를 나갔을 때 시스템 메시지를 확인할 수 있어야 한다.

1. 다른 플레이어가 접속을 종료하거나 다른 필드로 이동한다.
2. 채팅창에 "[플레이어명] left" 형태의 시스템 메시지가 표시된다.
3. 시스템 메시지는 일반 채팅 메시지와 다른 색상(회색 이탤릭체)으로 구분된다.

---

### [STORY 12-12] 채팅 메시지가 100개를 초과하면 가장 오래된 메시지부터 자동 삭제되어야 한다.

1. 채팅창에 메시지가 계속 쌓여 100개를 초과하게 된다.
2. 가장 오래된 메시지(화면 상단의 메시지)부터 순서대로 자동으로 삭제된다.
3. 항상 최신 100개의 메시지만 채팅창에 유지된다.
4. 유저가 별도로 조작할 필요 없이 자동으로 처리된다.

---

### [STORY 12-13] Normal 필드의 채팅 메시지는 Hardcore 필드 플레이어에게 표시되지 않아야 한다.

1. Normal 필드에 있는 플레이어 A가 메시지를 전송한다.
2. 해당 메시지는 Normal 필드에 있는 플레이어들에게만 전달된다.
3. Hardcore 필드에 있는 플레이어 B의 채팅창에는 해당 메시지가 표시되지 않는다.
4. 반대 방향도 동일하게 적용된다: Hardcore 필드의 채팅은 Normal 필드 플레이어에게 표시되지 않는다.
5. 각 필드의 채팅은 해당 필드에 있는 유저들에게만 전달되어 완전히 격리된다.
