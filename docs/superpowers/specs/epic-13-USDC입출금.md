# [Epic 13] USDC 입출금 서비스

> 작성 기준: v5.0 | 작성일: 2026-03-24
> 관점: 유저 플로우 (비개발자용)

---

## 목적

지갑을 연결한 유저가 Sepolia 테스트넷 USDC(6 decimals)를 게임 크레딧으로 충전(입금)하거나, 충전 크레딧을 USDC로 환급(출금)할 수 있는 서비스다.

---

## 입금 (Deposit)

### [STORY 13-1] 유저는 입금 탭을 열어 입금 화면을 확인할 수 있어야 한다.

1. 유저가 내 정보 팝업에서 '입금(Deposit)' 탭을 클릭한다.
2. 입금 탭에 다음이 표시된다:
   - USDC 금액 입력 필드 (정수만 입력 가능)
   - 금액 감소(-) / 증가(+) 버튼
   - 예상 수령 크레딧 미리보기 ("수령 예정: 0 크레딧")
   - '입금' 버튼

### [STORY 13-2] 유저는 입금 금액을 직접 입력하거나 +/- 버튼으로 조정할 수 있어야 한다.

1. 금액 입력 필드에 숫자를 입력한다. 숫자 이외의 문자(소수점 포함)는 자동 제거되어 정수만 유지된다.
2. '+' 버튼 클릭 시 1 증가, '-' 버튼 클릭 시 1 감소한다. (1 미만으로 내려가지 않음)
3. 금액 변경 시 예상 수령 크레딧이 즉시 갱신된다. (입력 USDC x 10,000)

### [STORY 13-3] 유저가 '입금' 버튼을 클릭하면 2단계 서명 절차가 진행되어야 한다.

1. 유저가 '입금' 버튼을 클릭한다.
2. 시스템이 사전 확인한다:
   - 입력값이 1 이상의 정수인지 확인. 아닐 경우 "최소 금액은 1 USDC입니다." 표시 후 중단.
   - 지갑이 연결되어 있는지 확인. 아닐 경우 "지갑을 먼저 연결해 주세요" 표시 후 중단.
   - Sepolia 테스트넷에 연결되어 있는지 확인. 아닐 경우 "Sepolia 테스트넷으로 네트워크를 전환해 주세요" 안내와 함께 전환 요청을 보낸다.
3. **Step 1 — Approve**: 버튼이 "승인 중..."으로 변경되고, 지갑에 USDC 사용 승인(ERC-20 approve) 서명 요청이 표시된다.
4. 유저가 승인 서명에 동의하면 버튼이 "승인 대기 중..."으로 변경되고, 온체인 확인을 기다린다.
5. **Step 2 — Deposit**: 승인 완료 후 버튼이 "입금 중..."으로 변경되고, 지갑에 vault deposit(`0xb6b55f25`) 서명 요청이 표시된다.
6. 유저가 입금 서명에 동의하면 버튼이 "확인 중..."으로 변경되고, "입금 트랜잭션이 전송되었습니다! N USDC -> N 크레딧" 성공 알림이 표시된다.

### [STORY 13-4] 온체인 입금 확인 후 충전 크레딧이 지급되어야 한다.

1. 블록체인에서 입금 처리가 완료된다.
2. 충전 크레딧이 즉시 잔액에 추가되고 화면 상단 잔액이 갱신된다.
3. 입금 금액 입력 필드와 예상 수령 크레딧이 0으로 초기화된다.
4. '입금' 버튼이 원래 상태로 복원된다.

### [STORY 13-5] 입금 과정 중 서명을 거부하면 크레딧 변동 없이 원래 상태로 복원되어야 한다.

1. Approve 단계에서 서명을 거부하면: 오류 메시지 표시 후 버튼 복원. 크레딧 변동 없음.
2. Deposit 단계에서 서명을 거부하면: 오류 메시지 표시 후 버튼 복원. 크레딧 미지급. (승인만 완료된 상태)

## 출금 (Withdraw)

### [STORY 13-6] 유저는 출금 탭을 열어 출금 화면을 확인할 수 있어야 한다.

1. 유저가 내 정보 팝업에서 '출금(Withdraw)' 탭을 클릭한다.
2. 출금 탭에 다음이 표시된다:
   - USDC 금액 입력 필드 (정수만 입력 가능)
   - 금액 감소(-) / 증가(+) 버튼
   - 차감될 크레딧 미리보기 ("차감 예정: 0 크레딧")
   - '출금' 버튼

### [STORY 13-7] 유저는 출금 금액을 직접 입력하거나 +/- 버튼으로 조정할 수 있어야 한다.

1. 금액 입력 필드에 숫자를 입력한다. 숫자 이외의 문자(소수점 포함)는 자동 제거되어 정수만 유지된다.
2. '+' 버튼 클릭 시 1 증가, '-' 버튼 클릭 시 1 감소한다. (1 미만으로 내려가지 않음)
3. 금액 변경 시 차감될 크레딧이 즉시 갱신된다. (입력 USDC x 10,500 / 수수료 5% 포함)

### [STORY 13-8] 유저가 '출금' 버튼을 클릭하면 서명 요청이 진행되어야 한다.

1. 유저가 '출금' 버튼을 클릭한다.
2. 시스템이 사전 확인한다:
   - 입력값이 1 이상의 정수인지 확인. 아닐 경우 "최소 금액은 1 USDC입니다." 표시 후 중단.
   - 지갑이 연결되어 있는지 확인. 아닐 경우 "지갑을 먼저 연결해 주세요" 표시 후 중단.
   - Sepolia 테스트넷에 연결되어 있는지 확인. 아닐 경우 전환 안내.
   - 보유 충전 크레딧이 차감될 크레딧 이상인지 확인. 부족하면 "크레딧이 부족합니다." 표시 후 중단.
3. 버튼이 "출금 중..."으로 변경되고 지갑에 vault withdraw(`0x2e1a7d4d`) 서명 요청이 표시된다.
4. 유저가 서명에 동의하면 "출금 트랜잭션이 전송되었습니다! N 크레딧 -> N USDC" 성공 알림이 표시된다.

### [STORY 13-9] 온체인 출금 확인 후 충전 크레딧이 차감되어야 한다.

1. 블록체인에서 출금 처리가 완료된다.
2. 충전 크레딧에서 해당 금액이 차감되고 화면 상단 잔액이 갱신된다.
3. 출금 금액 입력 필드와 차감 예정 크레딧이 0으로 초기화된다.

### [STORY 13-10] 출금 서명을 거부하면 크레딧은 차감되지 않아야 한다.

1. 유저가 지갑 팝업에서 출금 서명을 거부한다.
2. 오류 메시지 표시 후 버튼이 복원된다.
3. 크레딧은 차감되지 않는다.

## 공통 안전 정책

### [STORY 13-11] 입출금 진행 중에는 버튼이 비활성화되어 중복 요청을 방지해야 한다.

1. 버튼 클릭 즉시 비활성화 상태가 된다.
2. 트랜잭션이 완료되거나 오류가 발생한 후에만 버튼이 다시 활성화된다.

### [STORY 13-12] 동일 트랜잭션 해시로 크레딧을 중복 획득할 수 없어야 한다.

1. 이미 처리된 TX 해시로 재요청하면 "이미 처리된 트랜잭션입니다." 오류를 반환한다.
2. 서버는 처리 완료된 TX 해시를 in-memory Set으로 관리하여 중복을 차단한다.

### [STORY 13-13] 트랜잭션이 10분 이상 미확인 상태이면 타임아웃 안내를 표시해야 한다.

1. 트랜잭션 전송 후 10분이 경과해도 확인이 없으면 "트랜잭션 처리가 지연되고 있습니다. TX 해시로 직접 확인하세요." 메시지를 표시한다.
2. 버튼이 재활성화되어 유저가 다른 작업을 진행할 수 있다.
3. 서버는 해당 TX를 계속 모니터링하며, 이후 확인되면 크레딧을 자동 처리한다.

### [STORY 13-14] 온체인 트랜잭션 실패(revert) 시 즉시 알림을 표시해야 한다.

1. 트랜잭션 실패를 감지하면 "트랜잭션이 실패했습니다." 알림을 표시한다.
2. 입금 실패 시: 크레딧 미지급.
3. 출금 실패 시: 예약(lock)되었던 크레딧을 즉시 복구하고 잔액에 재반영한다.

### [STORY 13-15] 출금 요청 시 크레딧은 즉시 예약(lock)되어 다른 용도로 사용할 수 없어야 한다.

1. 출금 요청 접수 즉시 해당 금액을 잔액에서 예약 상태로 전환한다.
2. 예약된 크레딧은 곡괭이 구매, 추가 출금 등에 사용할 수 없다.
3. 온체인 확인 시 최종 차감. 실패 시 예약 취소 후 잔액 복구.

### [STORY 13-16] 인게임 크레딧(퀘스트 보상 등)은 출금할 수 없어야 한다.

1. 출금은 충전 크레딧(USDC 입금으로 획득한 크레딧)에서만 가능하다.
2. 인게임 크레딧은 출금 잔액에 포함되지 않는다.

### [STORY 13-17] 지갑 미연결 유저는 입출금을 시도할 수 없어야 한다.

1. 닉네임 로그인 유저가 입금 또는 출금 버튼을 클릭하면 "지갑을 먼저 연결해 주세요" 오류가 표시된다.
2. 트랜잭션은 전송되지 않는다.

### [STORY 13-18] "확인 중" 상태에서 팝업을 닫아도 서버는 트랜잭션 모니터링을 계속해야 한다.

1. 유저가 팝업을 닫거나 브라우저를 전환해도 서버 모니터링은 중단되지 않는다.
2. 트랜잭션 확인 시 크레딧이 자동 처리되고, 유저가 다시 팝업을 열면 완료 상태를 확인할 수 있다.

## 정책 요약

| 항목 | 값 |
|------|-----|
| 네트워크 | Ethereum Sepolia 테스트넷 |
| USDC | 6 decimals |
| 입금 환율 | 1 USDC = 10,000 크레딧 (충전 크레딧) |
| 출금 환율 | 10,500 크레딧 = 1 USDC (수수료 5%) |
| Approve 방식 | ERC-20 approve |
| Deposit 방식 | vault deposit (`0xb6b55f25`) |
| Withdraw 방식 | vault withdraw (`0x2e1a7d4d`) |
| TX 중복 방지 | in-memory Set |
| 입출금 최소 금액 | 1 USDC (정수만) |
| 타임아웃 | 10분 |
| 출금 가능 크레딧 | 충전 크레딧만 (인게임 크레딧 불가) |
