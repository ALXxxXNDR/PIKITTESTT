## ✦ [EPIC 13] USDC 입출금 서비스

### 목적
지갑을 연결한 유저가 USDC를 게임 크레딧으로 교환(입금)하거나, 게임 크레딧을 USDC로 교환(출금)할 수 있는 서비스다.

### 환율 정책
- 입금: 1 USDC → 10,000 크레딧 수령
- 출금: 1 USDC 수령 시 10,500 크레딧 차감 (5% 출금 수수료)

### 참고
- 네트워크: Ethereum Sepolia 테스트넷
- 지갑 연결 필수
- 인게임 크레딧(퀘스트 보상으로 획득한 크레딧)은 출금 불가

---

## 입금 (Deposit)

### [STORY 13-1] 유저는 입금(Deposit) 탭을 열어 입금 화면을 확인할 수 있어야 한다.

1. 유저가 내 정보 팝업에서 '입금(Deposit)' 탭을 클릭한다.
2. 입금 탭 내용이 표시된다:
   - USDC 금액 입력 필드 (정수만 입력 가능)
   - 금액 감소(-) 버튼
   - 금액 증가(+) 버튼
   - 예상 수령 크레딧 미리보기 ("수령 예정: 0 크레딧")
   - '입금' 버튼

---

### [STORY 13-2] 유저는 입금 금액을 직접 입력할 수 있어야 한다.

1. 유저가 금액 입력 필드에 숫자를 입력한다.
2. 숫자 이외의 문자(소수점 포함)는 자동으로 제거된다.
3. 정수만 유지된다.

---

### [STORY 13-3] 금액을 입력하면 수령할 크레딧이 실시간으로 표시되어야 한다.

1. 유저가 금액을 입력하는 즉시 예상 수령 크레딧이 갱신된다.
2. 계산식: 입력한 USDC × 10,000
3. 예시: 5 USDC 입력 시 → "수령 예정: 50,000 크레딧"

---

### [STORY 13-4] 유저는 +/- 버튼으로 금액을 1 USDC 단위로 조정할 수 있어야 한다.

1. '+' 버튼 클릭 시 현재 값에 1을 더한다.
2. '-' 버튼 클릭 시 현재 값이 1보다 크면 1을 뺀다. (1 미만으로 내려가지 않음)
3. 버튼 클릭 후 예상 수령 크레딧이 즉시 갱신된다.

---

### [STORY 13-5] 유저가 '입금' 버튼을 클릭하면 첫 번째 승인(Approve) 서명 요청이 표시되어야 한다.

1. 유저가 '입금' 버튼을 클릭한다.
2. 시스템이 다음을 확인한다:
   - 입력값이 양의 정수인지 확인. 아닐 경우 "유효한 금액을 입력해 주세요" 오류 메시지 표시 후 중단.
   - 지갑이 연결되어 있는지 확인. 아닐 경우 "지갑을 먼저 연결해 주세요" 오류 메시지 표시 후 중단.
3. 모든 확인 통과 시 버튼이 "승인 중..."으로 변경되고 클릭 불가 상태가 된다.
4. 지갑(MetaMask)에 USDC 사용 승인 서명 요청 팝업이 표시된다.

---

### [STORY 13-6] Approve 서명 후 온체인 처리 대기 상태가 표시되어야 한다.

1. 유저가 지갑에서 승인 서명에 동의한다.
2. 버튼이 "승인 대기 중..."으로 변경된다.
3. 온체인에서 승인 트랜잭션이 확인될 때까지 대기한다.

---

### [STORY 13-7] Approve 확인 완료 후 두 번째 입금(Deposit) 서명 요청이 표시되어야 한다.

1. 승인 트랜잭션이 온체인에서 확인된다.
2. 버튼이 "입금 중..."으로 변경된다.
3. 지갑에 입금 트랜잭션 서명 요청 팝업이 표시된다.

---

### [STORY 13-8] 입금 서명 후 처리 완료 대기 상태가 표시되어야 한다.

1. 유저가 입금 서명에 동의한다.
2. 버튼이 "확인 중..."으로 변경된다.
3. "입금 트랜잭션이 전송되었습니다! N USDC → N 크레딧" 성공 알림이 표시된다.

---

### [STORY 13-9] 서버가 온체인 입금을 확인하면 크레딧이 지급되어야 한다.

1. 블록체인에서 입금 처리가 완료된다.
2. 충전 크레딧이 즉시 잔액에 추가된다.
3. 화면 상단 잔액이 갱신된다.

---

### [STORY 13-10] 입금 완료 후 잔액이 즉시 갱신되고 입력 필드가 초기화되어야 한다.

1. 잔액 갱신이 화면에 반영된다.
2. 입금 금액 입력 필드가 0으로 초기화된다.
3. 예상 수령 크레딧도 "0 크레딧"으로 초기화된다.
4. 버튼이 다시 '입금' 상태로 복원된다.

---

### [STORY 13-11] Approve 단계에서 서명을 거부하면 오류 메시지가 표시되어야 한다.

1. 유저가 지갑에서 승인 서명을 거부한다.
2. 오류 메시지 알림이 표시된다.
3. '입금' 버튼이 원래 상태로 복원되고 다시 클릭 가능해진다.
4. 크레딧에는 아무런 변화가 없다.

---

### [STORY 13-12] Deposit 단계에서 서명을 거부하면 오류 메시지가 표시되어야 한다.

1. 승인(Approve)은 완료되었으나 유저가 입금(Deposit) 트랜잭션을 거부한다.
2. 오류 메시지 알림이 표시된다.
3. '입금' 버튼이 복원된다.
4. 크레딧은 지급되지 않는다. (승인만 완료된 상태)

---

### [STORY 13-13] 입출금 진행 중에는 버튼이 비활성화되어 중복 클릭을 방지해야 한다.

1. 버튼 클릭 즉시 비활성화 상태가 된다.
2. 트랜잭션이 완료되거나 오류가 발생한 후에만 버튼이 다시 활성화된다.

---

## 출금 (Withdraw)

### [STORY 13-14] 유저는 출금(Withdraw) 탭을 열어 출금 화면을 확인할 수 있어야 한다.

1. 유저가 내 정보 팝업에서 '출금(Withdraw)' 탭을 클릭한다.
2. 출금 탭 내용이 표시된다:
   - USDC 금액 입력 필드 (정수만 입력 가능)

   - 금액 증가(+) 버튼
   - 차감될 크레딧 미리보기 ("차감 예정: 0 크레딧")
   - '출금' 버튼

---

### [STORY 13-16] 금액을 입력하면 차감될 크레딧이 실시간으로 표시되어야 한다.

1. 유저가 금액을 입력하는 즉시 차감될 크레딧이 갱신된다.
2. 계산식: 입력한 USDC × 10,500 (5% 수수료 포함)

---

3. 버튼 클릭 후 차감될 크레딧이 즉시 갱신된다.

---

### [STORY 13-18] 유저가 '출금' 버튼을 클릭하면 서명 요청이 표시되어야 한다.

1. 유저가 '출금' 버튼을 클릭한다.
2. 시스템이 다음을 확인한다:
   - 입력값이 양의 정수인지 확인. 아닐 경우 오류 메시지 표시 후 중단.
   - 지갑이 연결되어 있는지 확인. 아닐 경우 "지갑을 먼저 연결해 주세요" 오류 메시지 표시 후 중단.
   - 현재 보유 크레딧이 차감될 크레딧 이상인지 확인. 부족하면 오류 메시지 표시 후 중단.
3. 모든 확인 통과 시 버튼이 "출금 중..."으로 변경되고 클릭 불가 상태가 된다.
4. 지갑에 출금 트랜잭션 서명 요청 팝업이 표시된다.

---

### [STORY 13-19] 출금 서명 후 처리 완료 대기 상태가 표시되어야 한다.

1. 유저가 서명에 동의한다.
2. "출금 트랜잭션이 전송되었습니다! N 크레딧 → N USDC" 성공 알림이 표시된다.

---

### [STORY 13-20] 서버가 온체인 출금을 확인하면 크레딧이 차감되어야 한다.

1. 블록체인에서 출금 처리가 완료된다.
2. 충전 크레딧에서 해당 금액이 차감된다.
3. 화면 상단 잔액이 갱신된다.

---

### [STORY 13-21] 출금 완료 후 잔액이 즉시 차감 반영되어야 한다.

1. 잔액 변경이 화면에 반영된다.
2. 출금 금액 입력 필드가 초기화된다.
3. 차감 예정 크레딧도 "0 크레딧"으로 초기화된다.

---

### [STORY 13-22] 잔액이 부족할 경우 출금 버튼 클릭 시 오류 메시지가 표시되어야 한다.

1. 보유 크레딧 < 차감될 크레딧인 경우
2. "크레딧이 부족합니다. N 크레딧이 필요합니다." 오류 메시지가 표시된다.
3. 트랜잭션은 전송되지 않는다.

---

### [STORY 13-23] 출금 트랜잭션을 거부하면 크레딧은 차감되지 않아야 한다.

1. 유저가 지갑 팝업에서 출금 트랜잭션을 거부한다.
2. 오류 메시지가 표시된다.
3. 버튼이 복원된다.
4. 크레딧은 차감되지 않는다.

---

### [STORY 13-24] 지갑 미연결 상태에서 입출금을 시도하면 오류 메시지가 표시되어야 한다.

1. 유저가 지갑 미연결 상태에서 입금 또는 출금 버튼을 클릭한다.
2. "지갑을 먼저 연결해 주세요" 오류 메시지가 표시된다.
3. 트랜잭션은 전송되지 않는다.

---

### [STORY 13-25] 서비스가 아직 준비되지 않은 경우 오류 메시지가 표시되어야 한다.

1. 유저가 입금 또는 출금을 시도한다.
2. 서비스가 준비되지 않은 경우 "서비스가 아직 준비되지 않았습니다." 오류 메시지가 표시된다.
3. 트랜잭션은 전송되지 않는다.

---

### [STORY 13-26] 소수점 금액 입력은 자동으로 정수로 변환되어야 한다.

1. 유저가 입력 필드에 소수점이 포함된 값(예: "1.5")을 입력한다.
2. 소수점 및 비숫자 문자가 자동으로 모두 제거된다.
3. 남은 값은 정수로만 구성된다.
4. 입력값이 비어 있거나 0인 경우 버튼 클릭 시 오류 메시지가 표시된다.

---

### [STORY 13-27] 동일한 트랜잭션을 재사용해 크레딧을 중복 획득할 수 없어야 한다.

1. 이미 처리가 완료된 트랜잭션으로 다시 입금 또는 출금을 요청한다.
2. 서버가 "이미 처리된 트랜잭션입니다." 오류 응답을 반환한다.
3. 크레딧이 중복으로 지급되거나 차감되지 않는다.
