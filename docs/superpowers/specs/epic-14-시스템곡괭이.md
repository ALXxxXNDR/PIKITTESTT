# [Epic 14] 시스템 곡괭이 서비스

> 작성 기준: v5.0 | 작성일: 2026-03-24
> 관점: 유저 플로우 (비개발자용)

---

## 목적

PIKIT 시스템 곡괭이는 항상 필드에 존재하는 하우스 전용 곡괭이다. 마젠타 색상으로 빠르게 낙하하며 넓게 튕기고, 블록을 적극적으로 파괴한다. 게임 진행 속도를 높이고 긴장감을 만드는 변수 역할을 한다.

---

### [STORY 14-1] 게임이 시작되면 PIKIT 곡괭이 1개가 항상 필드에 존재해야 한다.

1. 서버가 시작되면 필드에 마젠타(`#FF00FF`) 시스템 곡괭이 1개가 자동 스폰된다.
2. 플레이어가 없어도 항상 1개가 필드에 존재한다.
3. 빠르게 낙하하고 넓게 튕기며 블록을 적극적으로 파괴하는 모습을 유저가 볼 수 있다.

### [STORY 14-2] PIKIT 곡괭이가 블록을 파괴하면 보상은 누구에게도 지급되지 않아야 한다.

1. 시스템 곡괭이가 블록에 충돌하여 블록이 파괴된다.
2. 이 블록의 보상 크레딧은 어떤 플레이어에게도 지급되지 않는다.

### [STORY 14-3] PIKIT 곡괭이는 상점에 표시되지 않아야 한다.

1. 유저가 상점을 열면 구매 가능한 곡괭이 5종(Basic, Power, Light, Swift, Elite)만 표시된다.
2. 시스템 곡괭이는 목록에 없으며, 유저가 구매하거나 조작할 수 없다.

### [STORY 14-4] PIKIT 곡괭이는 일반 곡괭이와 시각적으로 구별되어야 한다.

1. 항상 마젠타(`#FF00FF`) 색상으로 표시된다.
2. 유저 곡괭이보다 1.2배 더 큰 크기로 렌더링된다.
3. 빠르게 낙하하며(gravity 0.8x) 넓게 튕긴다(speed 0.6x).
4. DMG 8. 수명 무한.
5. 툴팁에서 "PIKIT"으로 표시된다.

### [STORY 14-5] PIKIT 곡괭이가 필드에서 사라지면 5초 이내에 자동으로 다시 등장해야 한다.

1. 시스템 곡괭이가 어떤 이유로 필드에서 제거된다.
2. 5초 이내에 필드 상단에서 새로운 시스템 곡괭이가 다시 등장한다.
3. 항상 1개만 유지된다.

### [STORY 14-6] PIKIT 곡괭이가 잭팟 블록에 마지막 타격을 가하면 잭팟 풀은 유지되고 블록만 소멸해야 한다.

1. 시스템 곡괭이의 공격으로 잭팟 블록 체력이 0이 되면, 잭팟 풀은 플레이어에게 지급되지 않고 유지된다.
2. 잭팟 블록은 소멸하고 풀은 그대로 누적 상태를 유지한다.

### [STORY 14-7] 서버 재시작 후 시스템 곡괭이는 즉시 재생성되어야 한다.

1. 서버가 재시작되면 초기화 시점에 시스템 곡괭이를 즉시 스폰한다.
2. 재시작 전 상태(위치 등)는 복구되지 않으며 새로 시작한다.
3. 서버 재시작으로 인해 시스템 곡괭이가 영구적으로 사라지는 상태가 발생해서는 안 된다.

### [STORY 14-8] Normal 필드와 Hardcore 필드에는 각각 독립적인 시스템 곡괭이가 존재해야 한다.

1. 시스템 곡괭이는 각 필드(Normal, Hardcore)마다 독립적으로 1개씩 관리된다.
2. 한 필드의 시스템 곡괭이 상태가 다른 필드에 영향을 주지 않는다.

### [STORY 14-9] 하우스 엣지는 블록 확률로 확보하며, 시스템 곡괭이는 게임 변수 역할만 한다.

1. 하우스 엣지는 블록 보상 확률 설계(환수율 90%)로 확보한다.
2. 시스템 곡괭이는 하우스 엣지 확보 수단이 아니다.
3. 시스템 곡괭이의 역할은 게임 진행 속도를 높이고 블록 파괴 경쟁에서 긴장감과 변수를 만드는 것이다.

## 정책 요약

| 항목 | 값 |
|------|-----|
| 색상 | 마젠타 (`#FF00FF`) |
| 크기 | 1.2x |
| DMG | 8 |
| Gravity | 0.8x (빠른 낙하) |
| Speed | 0.6x (넓은 튕김) |
| 수명 | 무한 |
| 파괴 보상 | 미지급 (누구에게도) |
| 잭팟 블록 파괴 시 | 풀 유지, 블록만 소멸 |
| 상점 표시 | 미표시 (5종 유저 곡괭이만) |
| 재생성 | 사라진 후 5초 이내 |
| 필드별 관리 | Normal / Hardcore 각 1개 독립 |
| 서버 재시작 | 즉시 재생성 |
