## ✦ [EPIC 15] 하우스 수익 추적 및 잭팟 풀 관리

### 목적
플레이어 곡괭이 수명 만료 시점마다 하우스 수익을 집계하고, 일정 마일스톤마다 잭팟 풀에 자동 적립하는 시스템이다. 이를 통해 잭팟 보상이 하우스가 실제로 벌어들인 수익 내에서만 지급되어 하우스 손실이 발생하지 않는다.

---

### [STORY 15-1] 곡괭이 수명이 만료될 때마다 해당 곡괭이의 하우스 수익이 집계되어야 한다.

1. 플레이어의 곡괭이 수명이 만료된다.
2. 서버에서 해당 곡괭이의 하우스 수익을 계산한다:
   - 하우스 수익 = 곡괭이 구매가 - 해당 곡괭이로 획득한 총 보상
3. 하우스 수익이 0보다 크면 (하우스가 이익을 본 경우) 누적 하우스 수익 카운터에 추가된다.
4. 곡괭이가 손익분기를 넘어 플레이어 수익이 발생한 경우(0 이하) 카운터에 추가되지 않는다.

---

### [STORY 15-2] 누적 하우스 수익이 50,000 크레딧 달성할 때마다 잭팟 풀에 2,500 크레딧이 자동으로 추가되어야 한다.

1. 누적 하우스 수익 카운터가 50,000 크레딧 임계값에 도달한다.
2. 잭팟 풀에 **2,500 크레딧**이 자동으로 추가된다.
3. 누적 카운터에서 50,000이 차감된다. (초과분은 다음 마일스톤으로 이월)
4. 이 과정은 카운터가 여러 임계값을 동시에 초과할 경우 반복 처리된다.
5. 잭팟 풀 추가는 유저에게 별도로 알림이 가지 않는다. (내부 처리)

---

### [STORY 15-3] 잭팟 풀 현황은 게임 서버 브로드캐스트에 포함되어 클라이언트에 전달되어야 한다.

1. 서버는 매 브로드캐스트(초당 20회)마다 현재 잭팟 풀 금액을 게임 상태에 포함하여 전송한다.
2. 클라이언트는 이 정보를 수신하여 HUD에 실시간으로 표시한다.
3. 잭팟 풀이 0이면 표시되지 않거나 비활성화 스타일로 표시된다.

---

### [STORY 15-4] 서버가 재시작되면 마지막 저장 시점의 잭팟 풀과 하우스 수익 누적값을 이어간다.

1. 서버는 잭팟 풀 금액과 하우스 수익 누적 카운터를 주기적으로(매 1분) DB에 저장한다.
2. 서버 재시작 시 DB에서 마지막으로 저장된 잭팟 풀 금액과 카운터를 로드하여 이어간다.
3. 서버 재시작으로 잭팟 풀이 초기화되거나 하우스 수익 누적이 소실되어서는 안 된다.

---

### [STORY 15-5] 잭팟 블록 파괴 시 풀 초기화는 하우스 수익 시스템이 처리하며 중복 초기화를 방지해야 한다.

1. 잭팟 블록이 파괴되어 보상이 지급되면(Epic 8 처리), Epic 15의 잭팟 풀 카운터가 0으로 초기화된다.
2. 동시에 두 개의 초기화 이벤트가 처리되어 풀이 음수가 되거나 두 번 초기화되는 상황이 발생해서는 안 된다.

---

### [STORY 15-6] 동시에 여러 곡괭이가 수명 만료되어도 하우스 수익은 정확하게 집계되어야 한다.

1. 동일 서버 틱에 여러 곡괭이가 수명 만료되면 하우스 수익 계산(구매가 - 획득 보상)이 각 곡괭이에 대해 순차적으로 정확히 실행된다.
2. 동시 처리로 인해 하우스 수익이 중복 집계되거나 누락되어서는 안 된다.
3. 여러 곡괭이가 동시에 50,000 마일스톤을 통과한 경우, 통과한 횟수만큼 각각 2,500 크레딧씩 잭팟 풀에 적립한다.

---

### [STORY 15-7] 시스템 곡괭이가 파괴한 블록 보상은 별도 집계로 잭팟 풀에 직접 기여한다.

1. 시스템 곡괭이는 구매가가 없으므로 15-1의 "구매가 - 획득 보상" 방식을 적용할 수 없다.
2. 시스템 곡괭이가 파괴한 블록 보상 전액을 하우스 수익으로 간주하여 하우스 수익 누적 카운터에 직접 추가한다.
3. 이를 통해 시스템 곡괭이의 활동도 잭팟 풀 적립에 기여한다.

---

### 정책 요약

| 항목 | 값 |
|------|-----|
| 집계 시점 | 플레이어 곡괭이 수명 만료 시 |
| 적립 기준 | 하우스 수익 50,000cr마다 |
| 1회 적립량 | 2,500cr (하우스 수익의 5%) |
| 적립 방향성 | 하우스 실수익 기반 (플레이어 우위 런 제외) |
| 잭팟 지급 후 | 풀 0 초기화 |
| 잭팟 이탈 후 | 풀 유지 (계속 적립) |
