## v4.8 주요 변경 요약

**곡괭이 가격 (v4.8)**
| 곡괭이 | Normal | Hardcore |
|--------|--------|----------|
| Basic | 3,200cr | 32,000cr |
| Power | 8,300cr | 83,000cr |
| Light | 3,700cr | 37,000cr |
| Swift | 3,400cr | 34,000cr |

**잭팟 시스템 (v4.8)**
- 고정 보상 250,000cr 폐지 → 동적 풀
- 하우스 수익 50,000cr마다 잭팟 풀 +2,500cr 적립
- 풀 최소 50,000cr 이상일 때만 스폰 가능 (0.01%)
- 잭팟 블록 HP: 300 → 1,500 (레이드 필요)
- 화면 이탈 시 풀 유지, HP 회복, 재스폰 0.001%

**시스템 곡괭이 어댑티브 (v4.8)**
- 활성/휴식 2가지 모드
- 플레이어 곡괭이 ≤3개: 휴식 모드 (회색, 소형, 극저속)
- 최대 4개 동적 관리, 100명까지 지원
- HE 목표: 52~53% @ 2~30명, 54% @ 1명

## 작성 완료 기준

- [x] 모든 16개 Epic 파일 생성 완료
- [x] 각 파일에 기술 용어(소켓 이벤트명, 함수명, DOM ID 등) 없음
- [x] 모든 모달의 열기/닫기 시나리오 포함
- [x] 모든 에러/실패 시나리오 포함
- [x] 유저 행동 단위로 분해된 Step-by-Step 플로우 작성

## 작성 원칙

1. **비개발자 관점**: 소켓 이벤트명, 함수명, DOM ID 등 기술 용어 금지
2. **유저 행동 단위**: "유저가 X를 하면 Y가 된다" 형식
3. **완전한 시나리오**: 모든 버튼 클릭 결과, 모든 모달 열고/닫기, 모든 에러 상황 포함
4. **서식**:
   - Epic: `## ✦ [EPIC N] {이름}`
   - Story: `### [STORY N-M] 유저는 ~ 할 수 있어야 한다.`
   - 각 Story는 번호가 매겨진 단계별 플로우로 구성
